# LLM Wiki layout

This directory is the stable knowledge layer of the vault.

- `maps/` for entry points and MOCs
- `concepts/` for stable concept pages
- `topics/` for long-running topics
- `syntheses/` for cross-source synthesis

Current examples:
- `From User Story to Architecture`
- `Decision Modes for Engineering Work`
- `AI Engineering Workflow`
- `sources/` for structured source guides

The paired raw layer lives under `/raw` in the vault root.

## Canonical entrypoints

- [[wiki/index]] is the actual homepage
- [[wiki/NAMING]] defines naming and placement rules
- `wiki/Welcome.md` is legacy starter content and no longer acts as the home note

## Current rule of thumb

- old notes in `pages/`, `mobu/`, and `journals/` are gradually converted into source notes
- stable knowledge should be expressed in `wiki/`
- published content in `content/posts/` remains the publishing layer
- concept / topic / synthesis pages should follow the section grammar described in [[wiki/NAMING]]

The goal is to keep `wiki/` as the canonical knowledge layer rather than just another nested mini-vault.

## Legacy note

`wiki/Welcome.md` is kept temporarily only as historical starter content.
