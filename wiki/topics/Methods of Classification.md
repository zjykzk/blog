# Methods of Classification

分类方法的差异，本质上不是“分成几类”，而是**按什么维度切对象**。

同一堆对象，换一个维度切，得到的结构、判断和行动都会变。

## 六种常见分类方法

### 1. 属性分类

按对象“有什么特征”来分。

常见维度：
- 颜色
- 大小
- 材质
- 年龄
- 风格
- 学科归属

适合：
- 整理
- 检索
- 建目录

风险：
- 容易流于表面
- 看不到对象之间的关系

### 2. 功能分类

按对象“拿来做什么”来分。

比如不是问它是什么，而是问它在系统里承担什么作用。

适合：
- 流程设计
- 分工
- 工具系统
- 内容系统

风险：
- 一个对象可能同时承担多个功能
- 边界容易重叠

### 3. 结构层级分类

按对象处在哪一层来分。

常见切法：
- 现象 / 结构 / 激励 / 目标
- 战略 / 战术 / 执行
- 输入 / 处理 / 输出

适合：
- 复杂问题分析
- 避免不同抽象层混在一起

风险：
- 容易把网状关系硬切成塔状层级
- 层级感太强时，会忽略动态互动

### 4. 因果角色分类

按对象在因果结构里扮演什么作用来分。

常见切法：
- 原因 / 结果
- 条件 / 触发器
- 根因 / 放大器 / 症状
- 正反馈 / 负反馈

适合：
- 问题诊断
- 系统分析
- 找干预点

风险：
- 容易过度自信，以为自己已经看清因果
- 在复杂系统里，因果往往是闭环而不是单线

### 5. 时间分类

按对象处在什么时间位置或生命周期阶段来分。

常见切法：
- 过去 / 现在 / 未来
- 短期 / 中期 / 长期
- 起步期 / 增长期 / 成熟期 / 衰退期

适合：
- 规划
- 路线设计
- 节奏管理

风险：
- 容易把结构问题误看成阶段问题
- 时间切分很多时候是人为方便，不是对象本体属性

### 6. 可操作性分类

按“我能不能动它、怎么动它”来分。

常见切法：
- 可控 / 不可控
- 可逆 / 不可逆
- 高成本 / 低成本
- 高确定 / 低确定

适合：
- 决策
- 优先级判断
- 风险处理

风险：
- 过于功利，可能牺牲理解
- 今天不可控的，明天未必不可控

## 更深一层的几种方法

### 原型分类

不是看是否严格满足定义，而是看像不像一个典型成员。

适合自然语言和模糊概念。

风险是边界不清，也容易受刻板印象影响。

### 家族相似分类

对象不必共享一个共同本质，而是在许多局部特征上彼此相像。

它适合处理那种很难找到单一本质的概念。

风险是过松时，不利于行动。

### 二分法与多分法

- 二分法：快，适合决策
- 多分法：细，适合理解

风险是：二分法过粗，多分法过散。

### 分类树与分类矩阵

不是单维切分，而是组合多个维度。

- 分类树：逐级细分
- 分类矩阵：两个维度交叉，比如四象限

风险是维度一多，结构就会迅速失控。

## 怎么选方法

最关键的不是哪种方法“正确”，而是它是否适合当前任务。

可以问三句：

1. 我分类是为了什么？
2. 我真正关心的差异是什么？
3. 分完之后，能不能支持不同动作？

如果不能支持不同动作，这个分类大概率只是看起来整齐。

## 一句话压缩

> 分类方法的本质，不是怎么分，而是按什么维度分，才能让差异变得可操作。
