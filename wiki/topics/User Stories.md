# User Stories

用户故事的价值，不在于写出一个固定格式，而在于把“谁、想做什么、为什么”表达清楚，并让需求能继续往设计与实现层面流动。

## Common format

- As a（角色）
- I want to（功能 / 目标）
- So that（价值）

## Why it matters

一个好的用户故事应该帮助团队回答：

- 相关角色是谁
- 背景和问题是什么
- 输入、输出、流程和维护方式是什么
- 这项能力为什么值得做
