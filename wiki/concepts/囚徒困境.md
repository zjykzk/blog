# 囚徒困境

囚徒困境是一种博弈结构：两个行动者在不能互相信任、不能有效承诺、只能各自决策的条件下，都会选择对自己局部最优的行动，结果却落入对双方都更差的整体结局。

## What It Is

囚徒困境不是一个“人性自私”的道德故事，而是一个结构故事。它说明：即使参与者都足够聪明，只要信息结构、激励结构和承诺机制这样排列，坏结果也会被理性地生产出来。

经典形式是两个行动者都可以选择合作或背叛。对单个行动者来说，无论对方合作还是背叛，背叛都更安全或更有利；但当双方都背叛时，整体收益反而低于双方都合作。

```text
             B 合作        B 背叛
A 合作       R, R          S, T
A 背叛       T, S          P, P

T > R > P > S
T = 背叛诱惑
R = 双方合作收益
P = 双方背叛惩罚
S = 被背叛的最低收益
```

这个矩阵的关键不是四个数字，而是“背叛对个体占优，合作对整体更优”的断裂。

## How It Works

囚徒困境通常由四个结构条件共同生成：

1. 信息隔离：行动者不能可靠知道对方会怎么做。
2. 承诺缺失：行动者不能让“我会合作”变成可验证、可执行的承诺。
3. 背叛诱惑：如果对方合作，单方背叛能得到更高即时收益。
4. 被背叛风险：如果自己合作而对方背叛，自己承担最低收益。

因此，合作在整体上正确，在局部上危险。背叛在道德上可能更差，却在当前结构里更安全。

用 [[wiki/concepts/Mechanism Model]] 表达，囚徒困境可以压成一句机制句：在信息隔离和承诺缺失的约束下，背叛诱惑、被背叛风险和缺少惩罚通过局部理性选择，持续生成集体次优。

## Concept Cluster

囚徒困境和几个相邻概念一起构成合作机制的基本词汇：

- 纳什均衡：每个参与者都在给定他人选择时做出自己的最优反应；囚徒困境中的双方背叛就是一个稳定但低效的均衡。^[inferred]
- 公共品困境：公共收益依赖共同贡献，但个体可能更想搭便车。^[inferred]
- 集体行动问题：每个人都希望结果变好，却不愿单独承担先行动的成本。^[inferred]
- 信任博弈：关键不只是收益，还包括对他人行为和未来关系的预期。^[inferred]
- 重复博弈：当行动者知道未来还会再见，声誉、惩罚和回报会改变一次性背叛的收益结构。^[inferred]

这些概念都指向同一个问题：合作不是靠善意自然出现，而是由制度、重复关系、信息透明、声誉、惩罚能力和共同未来共同支撑。

## When to Use

囚徒困境适合用于分析那些“每个人都有局部理由，但系统结果一起变差”的场景：

- 团队成员都保留信息，避免暴露弱点，最后项目整体失败。
- 组织部门都优化自己的 KPI，最后跨部门协作成本上升。
- 企业都参与价格战，单个企业不降价会吃亏，全部降价后利润一起下降。
- 国家都扩军自保，结果安全感没有增加，军备压力却共同上升。
- 公共资源没人愿意先节制，最后资源被过度消耗。

使用这个概念时，不应只责备个体品德。更高杠杆的问题是：系统是否让合作足够安全？是否有沟通、承诺、惩罚、重复、透明和退出机制？

## Leverage Points

打破囚徒困境不是劝人“更善良”，而是重写合作的结构条件：

- 增加可见性：让行动、贡献、背叛和后果能被看见。
- 建立承诺机制：让合作承诺有成本、有记录、有执行路径。
- 引入重复关系：让一次性收益被长期声誉和未来回报约束。
- 设计惩罚与补偿：降低背叛收益，提高背叛成本，保护先合作者。
- 改变收益矩阵：让合作从整体正确变成局部也安全。
- 缩小试探单位：用小规模、低风险合作先建立信任。

从 [[wiki/concepts/System]] 视角看，囚徒困境暴露的是局部控制器和全局目标函数之间的失配：每个参与者都在优化自己的损失函数，但系统没有把全局合作收益编码进局部激励。^[inferred]

## Compact Model

```text
囚徒困境 = 信息隔离 + 背叛占优 + 合作风险 + 缺少承诺/惩罚/重复机制
```

最短的深句是：当系统不能让合作变得安全时，理性的人会一起制造出不理性的世界。

```text
          isolated actors
          /             \
     A guesses B     B guesses A
        |                 |
 local risk          local risk
 local incentive     local incentive
        \             /
         defect becomes safe
                 |
        collective outcome worsens
                 |
     redesign information + incentive
     + commitment + repetition + feedback
```
