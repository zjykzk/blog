# 可计算性模型

可计算性模型把“什么能被机械过程算出来”压成若干等价或分层的形式系统：上下文无关文法描述一类可识别的字符串结构，递归可枚举语言描述可被图灵机半判定的集合，μ-递归函数、λ 演算和图灵机则刻画同一类通用可计算过程。

## What It Is

形式语言里的“语言”是字符串集合，不是 Python、Java 这类具体编程语言。一个编程语言可以用上下文无关文法描述其程序文本的合法结构，同时它的执行语义可以是图灵完备的；前者回答“程序长什么样”，后者回答“程序能算什么”。

上下文无关文法也表达一种计算能力，但它对应的是下推自动机级别的识别能力。它能生成或识别上下文无关语言，例如 `a^n b^n`，但不能表达所有可计算过程，也不能识别需要更强约束的语言类。

递归可枚举语言是能被图灵机识别或枚举的语言。若字符串属于该语言，识别机最终会接受；若字符串不属于该语言，识别机可能拒绝，也可能永远运行。因此它也叫半可判定语言。

“递归”在这里是历史术语，接近“可计算的、机械可执行的”，不是编程语言里函数直接调用自己的狭义递归。现代教材也常把 recursively enumerable 改称 computably enumerable。

## How It Works

Chomsky hierarchy 可以粗略理解为一组语言类和机器模型的对应关系：

- 正则语言对应有限状态自动机。
- 上下文无关语言对应下推自动机。
- 递归可枚举语言对应图灵机。

递归函数理论从自然数函数出发。初始函数、复合和原始递归可以表达大量总会停机的计算，例如加法、乘法和幂；μ 最小化进一步允许“从 0 开始找最小的 y，使某个条件成立”。如果条件永远不成立，搜索永远不停。

μ-递归函数之所以与图灵机可计算函数等价，是因为两边能互相模拟。图灵机可以执行初始函数、复合、原始递归和 μ 搜索；反过来，μ-递归函数可以把图灵机配置编码成自然数，原始递归地模拟一步和 t 步运行，再用 μ 搜索最小停机时刻。

λ 演算用变量、函数抽象和函数应用表达计算：

```txt
term ::= x
       | λx. term
       | term term
```

它的核心计算规则是 β-归约：

```txt
(λx. M) N -> M[x := N]
```

λ 演算没有内置数字、布尔值、条件或循环，但可以用函数编码它们。Church numeral 把数字 n 编码为“把函数应用 n 次”；布尔值可以编码为二选一的函数；递归可以通过不动点组合子表达。

λ 演算和图灵机等价，也来自双向模拟。图灵机可以把 λ 表达式编码成符号树并机械执行 β-归约；λ 演算可以编码图灵机的状态、纸带、读写头和转移表，用递归反复应用 `STEP`，直到进入停机状态。

## When to Use

判断一个问题是否可计算，先把问题形式化为有限输入上的函数或 yes/no 判定问题。若能给出总会停机的算法，它就是可判定的；若只能在 yes 时迟早确认、no 时可能永远运行，它就是半可判定的；若能把停机问题规约到它，它就是不可判定的。

Rice 定理给程序语义性质一个快速判断：任何关于程序所计算函数或行为的非平凡性质都是不可判定的。代码文本是否出现某个字符通常可判定；程序是否永远返回 0、是否等价于另一个程序、是否对所有输入停机，通常落入不可判定区间。

经典计算模型本质上处理有限符号串。自然数函数并不狭窄，因为字符串、程序、证明、语法树、λ 表达式和图灵机配置都可以编码成自然数；但任意实数不能作为完整对象直接输入有限程序。

因此，这些模型能计算可计算实数，却不能计算任意实数。一个实数可计算，意思是存在程序给定精度 n 就能输出足够接近它的有理数近似，例如 π 和 e；大多数实数不可计算，因为程序只有可数多个，而实数不可数。
