# Code Expression for Human Understanding

面向人理解的代码表达，不是把代码逐行翻译成自然语言，而是把代码背后的意图、责任、约束、状态变化和失败模式显性化。

## What It Is

代码本身主要告诉机器“怎么做”；面向人的表达要补上“为什么这样做、它在系统里承担什么责任、它什么时候会失效”。

因此，好的代码表达应从语法动作上移到问题结构：

- 这段代码解决什么问题；
- 它负责保护哪条业务规则或工程约束；
- 它把什么输入转换成什么输出；
- 它改变了哪些状态，产生了哪些副作用；
- 它避免了什么失败模式；
- 它和上下游对象、模块、服务有什么关系。

逐行解释通常只能复述“做什么”。理解代码更需要看到它在系统中的责任位置。

## How It Works

### 先表达问题，而不是实现

代码说明的第一句最好不是“这里遍历列表、过滤、排序”，而是“这里要从候选任务中选出当前仍可执行的任务，并按最早截止时间优先处理”。

前者描述动作，后者描述意图。动作可以由代码呈现，意图通常需要表达补足。

### 把代码压缩成责任单元

人理解代码时，不会长期停留在每一行，而会把一组语句压缩为责任块。一个订单流程可以表达为身份确认、购买资格判断、资源占用、状态落库和后续通知，而不是只列出查用户、查商品、扣库存、写订单、发消息。

责任表达比步骤表达更接近软件设计，因为它说明每块代码为什么存在，以及边界在哪里。

### 用输入、转换、输出、副作用描述函数

函数可以被表达为一个小型转换器：在某些前提下，把输入状态转换成输出状态，同时可能读取、修改或通知外部世界。

一个稳定模板是：

> 这个函数在某个场景下，为了某个目标，基于某些约束，把某种输入转换成某种输出，同时避免某类失败模式。

这个模板让读者同时看到场景、目标、约束、状态变化和风险，而不是只看到语句顺序。

### 用业务原因解释分支

分支不应只解释为“如果 status == pending 就执行 A”。更有用的表达是：“只有待处理任务还允许取消；已执行或完成的任务不能再回滚，所以这里区分 pending 和非 pending。”

代码里的分支往往来自状态、权限、生命周期、异常路径、性能路径、历史兼容或不可信输入。表达分支时，应说明这个判断保护了什么边界。

### 用模型含义解释数据结构

字段列表本身不能让人理解模型。`User.id`、`name`、`role`、`createdAt` 应分别解释为稳定身份、展示名称、权限边界和审计/生命周期信息。

数据结构是系统对现实对象的建模方式。表达数据结构时，应区分身份字段、状态字段、权限字段、审计字段、展示字段、冗余字段和缓存字段。

### 用主轴承载复杂代码

复杂代码应先给主轴，再进入局部细节。主轴说明整体在做什么；细节说明关键规则、边界情况和容易出错的位置。

没有主轴，细节越多越乱；只有主轴，没有边界细节，又会变成空泛概括。

### 注释解释为什么，不重复做什么

注释不应重复“遍历用户列表”。注释更适合解释为什么不能取第一个管理员、为什么要兼容历史数据、为什么这段逻辑不能简化、为什么这里必须保持幂等。

代码已经能表达许多“做什么”；注释应补充代码本身难以表达的背景、约束和设计理由。

## UML and Text

结构化文字与 UML 不是替代关系，而是互补关系。

结构化文字适合表达：

- 为什么这么设计；
- 代码承担什么责任；
- 这个分支或异常处理为什么存在；
- 有什么业务约束、历史包袱或失败后果；
- 什么时候这个实现不再成立。

UML 和 C4 / Mermaid 图更适合表达：

- 有哪些对象、模块、服务或外部系统；
- 谁依赖谁，谁调用谁；
- 一个场景中的消息顺序；
- 一个对象的状态生命周期；
- 系统边界、部署边界和运行时拓扑。

简化判断是：读者卡在“为什么”时，用结构化文字；读者卡在“关系看不清”时，用图。真实工程文档通常先用文字解释意图和约束，再用图展示对象、模块、状态或交互关系。

UML 的风险是形式感压过信息密度，或者画出很细却很快过期的实现细节。结构化文字的风险是啰嗦、线性负担重，难以一眼展示复杂拓扑。两者都应服务理解，而不是成为文档仪式。

## When to Use

函数级解释通常优先使用结构化文字，因为重点是意图、前提、转换和失败模式。

领域模型解释适合使用文字加类图：文字说明对象的责任和建模理由，类图展示字段、关系和边界。

跨服务调用适合使用序列图加少量文字：图展示消息顺序，文字补充幂等、重试、补偿和责任边界。

状态生命周期适合使用状态图加文字：图展示合法迁移，文字说明迁移条件、不变量和禁止路径。

架构边界适合使用 C4、组件图或部署图加文字：图展示系统结构，文字说明设计取舍、质量属性和演化约束。
