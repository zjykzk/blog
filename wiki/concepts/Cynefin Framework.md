# Cynefin Framework

Cynefin Framework 适合用来判断问题处在哪种上下文里，再决定应该用什么动作模式，而不是先假定所有问题都能靠一种统一方法解决。

## Five contexts

### Obvious

- 因果关系清楚
- 存在最佳实践
- 更适合 `sense -> categorize -> respond`
- 典型场景：告警手册、标准化操作

### Complicated

- 有多个可能答案
- 问题可分析，但需要专家判断
- 更适合 `sense -> analyze -> respond`
- 目标是把问题进一步化简

### Complex

- 不知道自己不知道什么
- 需要实验来发现模式
- 更适合 `probe -> sense -> respond`
- 适合创新探索和高不确定性问题

### Chaotic

- 首要目标是止损
- 先行动，再恢复秩序
- 更适合 `act -> sense -> respond`
- 典型场景：线上事故、紧急回滚

### Disorder

- 当前还无法判断问题属于哪一类
- 需要先收集信息、拆分问题、再分类

## Practical use

这个框架最有用的地方，不是记住五个象限，而是避免误判：

- 不要用最佳实践去处理 complex 问题
- 不要在 chaotic 场景里长时间分析而不行动
- 不要把 disorder 当成已经分类好的问题
