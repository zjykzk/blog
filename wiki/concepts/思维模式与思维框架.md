# 思维模式与思维框架

思维模式是人长期稳定的默认认知倾向，决定人默认如何理解问题；思维框架是人为搭建的分析结构，帮助人有步骤、有维度地拆解某类问题。

一句话区分：**思维模式是怎么看；思维框架是怎么拆。**

## What It Is

思维模式关注“你倾向于怎么想”。它像默认操作系统，决定人遇到问题时优先看见什么、忽略什么、相信什么因果、采取什么反应。

典型例子包括：

- **线性思维模式**：默认寻找 A 导致 B、B 导致 C 的单线因果。
- **系统思维模式**：默认把问题放进要素、关系、反馈、延迟、目标和结构中理解。
- **成长型思维模式**：默认认为能力可以通过训练、反馈和方法改进。
- **固定型思维模式**：默认把失败解释为天赋不足或能力定型。

思维框架关注“你用什么结构来分析问题”。它像打开的分析软件，可以按问题类型切换。

典型例子包括：

- **5 Why**：连续追问为什么，逼近根因。
- **SWOT**：从优势、劣势、机会、威胁四个维度看局势。
- **第一性原理**：拆到基本事实，区分事实与假设，再从零设计。
- **MECE**：分类时要求不重叠、不遗漏。

## How It Works

思维模式更底层、更习惯化、更长期稳定；思维框架更工具化、更方法化、更可切换。两者不是并列工具，而是上下层关系：

| 维度 | 思维模式 | 思维框架 |
|---|---|---|
| 本质 | 默认认知倾向 | 分析工具结构 |
| 作用 | 决定怎么看世界 | 帮助怎么拆问题 |
| 稳定性 | 相对稳定 | 可随问题切换 |
| 例子 | 系统思维、成长型思维、批判性思维 | 5 Why、SWOT、MECE、第一性原理 |
| 类比 | 操作系统、内功 | 应用软件、招式 |
| 训练方式 | 长期改变认知习惯 | 学会具体方法并反复使用 |

思维模式会影响框架选择。系统思维模式更容易调用因果回路图、反馈环、瓶颈分析和全链路分析；批判性思维模式更容易调用证据-结论框架、假设检验、反例分析和论证结构分析；产品思维模式更容易调用用户旅程、Jobs To Be Done、AARRR 和价值链分析。

思维框架也会反向训练思维模式。反复使用 5 Why，人会更习惯追根因；反复使用 MECE，人会更习惯结构化；反复使用系统图，人会更习惯看关系而不是孤立点。

因此，思维模式是长期形成的认知肌肉，思维框架是训练这块肌肉的动作。

## Software Engineering Example

线上接口延迟突然升高时，线性思维模式可能先寻找一个最明显原因：是不是数据库慢了，所以先查数据库。

系统思维模式会先把延迟升高看成系统现象：入口流量、线程池、GC、数据库、缓存、下游、发布变更、限流和队列积压都可能通过关系与反馈共同生成结果。

随后才进入具体框架：

- 按调用链拆：客户端、网关、服务、缓存、数据库、下游。
- 按资源拆：CPU、内存、IO、网络、连接池。
- 按时间拆：发布前、发布中、发布后。
- 按指标拆：QPS、RT、错误率、饱和度。

完整过程是：思维模式决定是否从系统角度看问题；思维框架帮助把系统问题拆清楚。

## When to Use

这个区分适合用来诊断两类常见问题：

- **只有框架，没有模式**：会套 SWOT、MECE、5 Why，但底层仍急于下结论、只找单因果，容易机械套模板。
- **只有模式，没有框架**：知道“要系统思考”或“要批判性思考”，但没有可执行步骤，容易停留在口号和感觉。

更好的状态是：用正确的思维模式选择框架，用具体框架训练思维模式。
