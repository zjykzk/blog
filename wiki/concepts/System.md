# System

A system is a bounded set of interacting elements whose relations and feedback generate whole behavior over time.

## What It Is

A system is not merely a collection. A collection means things are colocated or grouped; a system means the elements affect one another through relations, boundaries, flows, and feedback.

The core terms inside the concept are element, relation, boundary, function, feedback, and whole. Without elements, the system has no carrier; without relations, it is only a pile; without a boundary, it cannot be identified; without feedback, it cannot regulate, amplify, stabilize, or evolve.

## How It Works

A minimal system can be expressed as `S = (E, R, B, F, T)`: elements, relations, boundary, feedback rules, and time. This expression is useful because it separates the parts from the pattern that makes the parts behave as a whole.

The expression becomes incomplete in strongly subjective, political, or interpretive settings. In those settings, who draws the boundary, how relations are interpreted, and which feedback is allowed to count are themselves part of the system. ^[inferred]

System behavior often appears as recurring results rather than isolated events. The durable question is not only “which part caused this?” but “which relations and feedback loops keep generating this result?” That is why system analysis connects directly to [[wiki/topics/Thinking in Systems]], [[wiki/concepts/Feedback Loops]], and [[wiki/concepts/Leverage Points]].

## When to Use

Use the concept of system when results recur, when local actions create nonlocal consequences, or when changing one element fails to change the pattern. It is especially useful for organizations, products, learning, ecological settings, software systems, AI agents, and family or social dynamics.

The concept is dangerous when it becomes too cold. Seeing structure can reduce blame and reveal leverage, but it can also flatten people into nodes and underweight pain, agency, and dignity. ^[inferred]

Two metaphors compete inside system thinking. The machine metaphor emphasizes inputs, outputs, parts, and control; it helps with engineering clarity but can hide history, adaptation, and meaning. The ecology metaphor emphasizes relations, redundancy, recovery, and co-evolution; it helps with living complexity but can make deliberate intervention feel less precise.

## Compact Model

System = elements + relations + boundary + feedback + whole behavior through time.

The simplest deep sentence is: a system is not a pile of things, but a set of relations that repeatedly generates a result.

```text
        boundary
   +----------------+
   |                |
   |  e1 -- r -- e2 |
   |   \        /   |
   |    r      r    |
   |     \    /     |

   |        |       |
   |      feedback  |
   |        |       |
   +--------v-------+
            time

          result
```
