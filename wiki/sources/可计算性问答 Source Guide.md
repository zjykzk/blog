# 可计算性问答 Source Guide

> Source: conversation:2026-06-17

## Capture Policy

这页保存一段围绕形式语言、递归函数、λ 演算和图灵机的中文教学问答。它是 source 层材料，保留问题推进顺序和解释结构；抽象后的稳定概念页是 [[wiki/concepts/可计算性模型]]。

这页不试图替代计算理论教材。它保存的是一条学习路径：从“编程语言能用 CFG 表示但又图灵完备”这个疑问出发，逐步分清语法识别能力、半判定能力、通用计算能力和有限符号系统的实数边界。

## What It Covers

这段问答覆盖七个连续问题：

1. CFG 表示编程语言、编程语言又图灵完备，这句话是什么意思？
2. CFG 是否也表达一种计算能力？
3. 什么是递归可枚举语言，为什么它对应图灵机？
4. 为什么叫递归，什么是递归函数？
5. 为什么“可能一直找下去”的 μ 最小化很重要？
6. λ 演算是什么，为什么它和图灵机等价？
7. 这些经典模型是否不能计算任意实数，以及如何判断一个问题是否可计算？

## Preserved Content

### CFG 和图灵完备处在不同层次

编程语言的语法通常可以用上下文无关文法描述；编程语言的执行语义则可能是图灵完备的。

这一区分可以压成：

```text
CFG 描述程序文本长什么样。
图灵完备描述程序执行时能算什么。
```

因此，“编程语言可用 CFG 表示”和“编程语言图灵完备”并不矛盾。前者是 parser 和形式语法层面的问题，后者是语义执行和计算能力层面的问题。

### CFG 也有计算能力，但不是通用计算

CFG 确实表达一种计算能力：它能生成或识别上下文无关语言。对应机器模型是下推自动机，它比有限状态自动机强，因为它有一个栈。

典型例子是：

```text
a^n b^n
```

它要求记住前面有多少个 `a`，再匹配同样数量的 `b`。有限状态自动机做不到，但下推自动机可以做到。

不过 CFG 不能表达任意计算，也不能覆盖所有语言约束。它和图灵机不在同一能力层级。

### 递归可枚举语言是半可判定语言

递归可枚举语言是能被某台图灵机识别的字符串集合。若输入属于该语言，图灵机最终停机并接受；若输入不属于该语言，图灵机可能拒绝，也可能永远不停机。

这就是半可判定：

```text
yes 时迟早能确认。
no 时不保证能确认。
```

停机问题的 yes 集合是典型例子：

```text
HALT = { <P, x> | 程序 P 在输入 x 上会停机 }
```

可以模拟 `P(x)`：如果它停机，就接受；如果它不停机，模拟过程也永远继续。

### “递归”是可计算性理论的历史术语

递归可枚举里的“递归”不是狭义的函数自调用，而是历史上 recursive function 和 computable function 的语境。早期递归函数理论、λ 演算和图灵机分别从自然数函数、函数替换和符号机器出发，后来被证明刻画同一类有效可计算过程。

因此，这里的 recursive 更接近：

```text
可计算的、机械可执行的。
```

现代教材常把 recursively enumerable 改称 computably enumerable，以减少这个误会。

### 递归函数从原始递归走向 μ-递归

递归函数理论从几个初始函数出发：

- 零函数。
- 后继函数。
- 投影函数。

再用复合和原始递归构造新函数。原始递归能表达加法、乘法和幂等大量总会停机的计算。

关键扩展是 μ 最小化：

```text
f(x) = μy [g(x, y) = 0]
```

它表示从 `y = 0` 开始找最小的 y，使条件成立。如果永远找不到，计算就永远不停。这一步让递归函数理论能表达真实程序里的无界搜索和可能不停止的计算。

### μ-递归函数和图灵机等价

等价来自双向模拟。

图灵机可以计算 μ-递归函数，因为它能执行初始函数、复合、原始递归和无界搜索。

μ-递归函数也能模拟图灵机，因为图灵机配置可以编码成自然数：

```text
配置 = 编码(状态, 读写头位置, 纸带内容)
```

一步转移可以用原始递归表达；运行 t 步可以用原始递归表达；寻找最小停机时刻则用 μ 最小化表达：

```text
T(x) = μt [config(t, x) 是停机配置]
```

这就是“图灵机的一步转移 + 重复 t 步 + 搜索停机时间”在递归函数理论中的对应结构。

### λ 演算用函数抽象和应用表达计算

λ 演算只有三种表达式：

```text
term ::= x
       | λx. term
       | term term
```

核心计算规则是 β-归约：

```text
(λx. M) N -> M[x := N]
```

λ 演算没有内置整数、布尔值、条件和循环，但可以用函数编码它们。Church numeral 把自然数 n 表示成“把函数应用 n 次”；布尔值可以表示为二选一；递归可以用不动点组合子表达。

### λ 演算和图灵机等价

图灵机可以模拟 λ 演算，因为 β-归约是有限、明确、机械的符号变换。图灵机可以把 λ 表达式编码成字符串或树，然后不断查找可归约表达式、替换变量、必要时重命名以避免变量捕获。

λ 演算也可以模拟图灵机。它可以编码状态、纸带左侧内容、当前格子、纸带右侧内容和转移表，再定义：

```text
STEP config = next_config
RUN config = if halted(config) then output(config) else RUN (STEP config)
```

递归部分通过不动点组合子接上。因此，图灵机和 λ 演算只是用不同符号结构实现同一类机械计算。

### 经典模型处理有限符号串

图灵机、λ 演算和 μ-递归函数本质上处理有限符号对象。自然数函数并不狭窄，因为程序、字符串、证明、语法树、图灵机配置和 λ 表达式都可以编码成自然数。

但完整的任意实数通常包含无限信息，不能直接作为有限输入交给图灵机。经典模型能计算可计算实数：给定精度 n，程序能输出足够接近该实数的有理数近似。π、e 和 `sqrt(2)` 是可计算实数；大多数实数不是可计算实数，因为程序可数而实数不可数。

### 判断可计算性的方法

判断一个问题是否可计算，先把它形式化为有限输入上的函数或 yes/no 问题。

常用路径有三条：

1. 给出一个总会停机的算法，则问题可判定。
2. 若只能在 yes 时找到证据，而 no 时可能永远找不到，则问题半可判定。
3. 若能把停机问题规约到该问题，则该问题不可判定。

Rice 定理提供程序语义性质的快速判断：任何关于程序计算出来的函数或行为的非平凡性质都是不可判定的。代码文本是否包含某个字符是语法性质，通常可判定；程序是否永远返回 0、是否对所有输入停机、是否和另一个程序等价，是语义性质，通常不可判定。
