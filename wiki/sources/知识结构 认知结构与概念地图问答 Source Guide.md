# 知识结构、认知结构与概念地图问答 Source Guide

> Source: 本次会话关于“认知结构、知识结构、概念地图”的中文教学问答

## Capture Policy

这页保存的是一次 conversational teaching source artifact。它保留问答中逐步形成的区分、分类、工具判断和最小模板，不直接合并进 [[wiki/sources/知识结构概念解剖 Source Guide]]，也不新建过大的稳定 concept 页。

## What It Covers

这段问答围绕三个问题展开：认知结构有哪些类型；知识结构有哪些类型，它与认知结构是什么关系；概念地图是不是捕获知识结构和认知结构的好工具。

核心结论是：知识结构偏向“知识之间怎样连接、分层、分类、因果化和模型化”；认知结构偏向“人如何注意、解释、判断、行动、复盘和更新”。概念地图非常适合捕获知识结构；如果加入判断、偏误、反馈和更新规则，它也可以成为捕获认知结构的入口。

这个 source guide 与 [[wiki/concepts/Knowledge as Network]]、[[wiki/concepts/Knowledge Types]]、[[wiki/concepts/Understanding]]、[[wiki/topics/Mental Models]] 和 [[wiki/sources/表征 图式 心智模型和解释框架 Source Guide]] 相邻。

## Preserved Content

### 认知结构的类型

认知结构不是单一东西，而是一组组织经验、解释信息、产生判断和调度行动的结构。

- **概念结构**：把世界分成不同概念和类别。例如知道“公司”“利润”“现金流”“估值”分别是什么。
- **分类结构**：知道事物属于哪一类、和谁相似、和谁不同。例如把企业分成消费品、平台、制造业、资源品。
- **因果结构**：理解“什么导致什么”。例如利率上升、融资成本上升、高估值资产承压之间的链条。
- **层级结构**：区分底层原则、中层模型和表层现象。例如商业模式、收入模式和促销活动之间的层级。
- **关系网络结构**：理解多个概念之间的连接。例如用户增长、留存、转化率、客单价、毛利率如何共同影响利润。
- **模型结构**：用稳定框架解释一类问题。例如供需模型、复利模型、飞轮模型、博弈模型和反馈回路模型。
- **脚本结构**：对常见情境有默认处理流程。例如遇到问题时按“现象 -> 原因 -> 约束 -> 方案 -> 反馈”处理。
- **叙事结构**：用故事线理解世界。例如把失败解释为“我不行”，或解释为“模型还没校准”。
- **价值结构**：决定什么重要、什么不重要。例如优先看效率、安全、长期复利或自由度。
- **元认知结构**：理解自己的理解方式。例如知道自己什么时候在过度概括、确认偏误或情绪化判断。

```text
认知结构 = 概念 + 分类 + 因果 + 层级 + 关系 + 模型 + 脚本 + 价值 + 自我校正
```

其中最关键的通常不是单个概念结构，而是因果结构、层级结构和元认知结构，因为它们决定一个人能否从信息里长出判断。

### 知识结构与认知结构的关系

知识结构是认知结构的一部分，但不是全部。

```text
认知结构 = 知识结构 + 解释结构 + 判断结构 + 行动结构 + 校正结构
```

知识结构回答“我知道什么，这些知识如何分类、分层、连接”；认知结构回答“我如何看见问题、解释事实、判断轻重、做出行动、根据反馈修正自己”。

以投资为例，知识结构可能包括会计、金融、商业和心理学知识；认知结构还包括如何判断公司是真赚钱还是账面赚钱，如何区分短期波动和长期价值，遇到亏损时如何归因，以及是否能识别确认偏误、过度自信和路径依赖。

因此，知识结构像图书馆或仓库；认知结构更像操作系统。知识结构解决“存了什么”，认知结构解决“怎么调用、解释、判断、行动、更新”。

一个人可以知识很多但认知结构差：会背概念，但不能判断真实问题。一个人也可能知识量暂时不大但认知结构好：能快速抓住关键变量，知道该补什么知识、怎么验证、怎么修正。

```text
好的知识结构 x 好的认知结构 = 可迁移的能力
```

### 知识结构的类型

知识结构描述知识材料如何被组织成可调用、可迁移、可推理的形状。

- **树状结构**：从总到分、从上到下组织知识。适合学科体系、课程目录和系统学习。
- **网状结构**：知识点之间互相连接，不只有上下级。适合复杂系统、经济、商业和社会问题。
- **链条结构**：按前后顺序或因果顺序组织。适合流程、因果推理、历史演化和业务分析。
- **层级结构**：区分底层原则、中层模型和表层案例。适合把经验提升为方法。
- **模块结构**：把知识拆成相对独立、可组合的模块。适合技能学习、工程、产品和管理。
- **矩阵结构**：用两个或多个维度交叉组织知识。适合分类、比较、定位和策略判断。
- **地图结构**：把一个领域画成空间地图，知道哪里是什么、彼此距离多远。适合进入新领域和建立全局感。
- **问题结构**：围绕问题组织知识，而不是围绕学科。适合真实世界问题和跨学科学习。
- **案例结构**：用典型案例承载知识。适合商业、历史、管理、法律和医学等经验密集领域。
- **模型结构**：围绕核心模型组织知识。适合形成可迁移判断力。
- **时间结构**：按历史演化组织知识。适合理解概念为什么出现、理论如何变形。
- **金字塔结构**：上层是结论，中层是理由，底层是证据。适合写作、汇报、咨询和表达。

```text
知识结构 = 树 + 网 + 链 + 层 + 模块 + 问题 + 模型
```

树状结构帮人有系统，网状结构帮人看连接，层级结构帮人分主次，模型结构帮人迁移，问题结构帮人解决真实问题。真正高级的知识结构通常不是单一形状，而是几种结构叠在一起。

### 捕获知识结构和认知结构的工具分工

捕获知识结构和捕获认知结构需要不同工具面。

知识结构偏向外部知识组织，可以用 Obsidian 双链、标签、属性和 graph，Obsidian Canvas 或 Excalidraw 画树、链、因果、地图和模型，Heptabase 这类白板式卡片系统做视觉拼图，个人 wiki 或 LLM Wiki 保存稳定概念、主题、来源和综合。

认知结构偏向思考过程、判断模型和反馈记录，可以用 Logseq 这类 block outliner 记录每日问题、推理链和想法流，用 Tana 这类结构化 outliner 保存判断模板、决策复盘和问题模板，也可以用判断日志、复盘模板、预测记录和假设验证表。

对当前 wiki 来说，最小可用组合是：

```text
Logseq/日记捕获原始思考
  -> Obsidian wiki 沉淀稳定知识
  -> Canvas 画结构
  -> 模板记录认知复盘
```

一个用于捕获认知结构的最小模板是：

```text
问题：
我看到的现象：
我调用的知识：
我使用的模型：
关键变量：
因果链：
我的判断：
可能的盲点：
反馈结果：
需要更新的认知结构：
```

### 概念地图的适用边界

概念地图是捕获知识结构的强工具，也是捕获认知结构的入口工具。

它天然适合表达概念有哪些、概念之间是什么关系、哪些是上位概念和下位概念、哪些关系是因果/组成/对比/依赖、一个领域的整体骨架是什么。

```text
认知结构
  -> 包含 -> 知识结构
  -> 决定 -> 信息如何被解释
  -> 依赖 -> 元认知
  -> 影响 -> 判断与行动
```

但概念地图只能部分捕获认知结构，因为认知结构还包括注意力模式、默认解释框架、重要性判断、冲突证据修正、盲点、防御、偏好、价值排序和行动反馈。

概念地图最擅长捕获概念结构、分类结构、关系结构、模型结构和问题结构；它不太擅长直接捕获情绪偏差、判断习惯、注意力模式、价值排序、反馈修正过程和真实行动中的选择模式。

如果要用概念地图同时捕获知识结构和认知结构，可以给每个节点增加几类边：

```text
是什么        is-a
由什么组成    part-of
导致什么      causes
我如何判断    judged-by
如何更新      updated-by
```

例如：

```text
投资判断
  -> 依赖 -> 企业现金流
  -> 依赖 -> 竞争优势
  -> 受到影响 -> 利率
  -> 可能偏误 -> 叙事迷恋
  -> 需要反馈 -> 买入后假设验证
  -> 更新规则 -> 若核心假设被证伪则重估
```

这样，概念地图就不只是知识图谱，而开始变成认知结构图。
