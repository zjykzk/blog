# Application Concept Anatomy Source Guide

This page preserves the knowledge extracted from the `ljg-learn` concept-anatomy note on [[wiki/concepts/Application]].

## Source Claim

The source defines application as putting knowledge, method, tool, rule, or capability into a concrete situation so it can produce actual effect.

It explicitly warns against two reductions: application is not mechanical execution, and it is not a lower-status activity beneath theory. Application is the interface where theory and reality check and reshape each other.

## Useful Distinctions

- Application versus possession: owning or explaining a concept is not yet application.
- Application versus execution: execution follows a procedure; application adapts a structure to a situation.
- Landing metaphor versus connection metaphor: "landing" implies theory descends into practice, while "connection" highlights coupling, feedback, and mutual transformation. ^[inferred]
