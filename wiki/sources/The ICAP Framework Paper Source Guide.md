# The ICAP Framework Paper Source Guide

> Source: Michelene T. H. Chi & Ruth Wylie (2014), “The ICAP Framework: Linking Cognitive Engagement to Active Learning Outcomes”, Educational Psychologist, 49(4), 219–243. DOI: 10.1080/00461520.2014.965823. PDF: https://dunkin.eeb.ucsc.edu/images/documents/The_ICAP_Framework_Linking_Cognitive_Engagement_to_Active_Learning_Outcomes.pdf

## What It Covers

这篇论文要解决的不是“主动学习是否比被动学习好”这个粗问题，而是：当教育研究和课堂实践都在说 active learning 时，究竟哪些外显学习行为代表了不同层级的认知参与，它们为什么可能带来不同学习结果。

论文提出 ICAP framework：学习者与材料接触时的外显行为可以分成四种模式：

- **Passive**：receiving，接收信息。
- **Active**：manipulating，操作或标记已有信息。
- **Constructive**：generating，生成材料中没有的新信息。
- **Interactive**：dialoguing，在对话中共同生成、挑战、修正和扩展理解。

论文的主假设是：

> Interactive > Constructive > Active > Passive

这个排序不是简单的课堂活动偏好，而是一个关于知识变化过程的假设：越高层的参与模式越可能引发更深的知识整合、推理、迁移与共同创造。

## Preserved Content

### 1. 论文的出发点：active learning 太粗，教师缺少可操作判据

论文开头指出，教育者普遍承认学生可以从被动接收中学习，但通过 active learning 往往学得更好。问题是，active learning 在文献和实践中经常被定义为“认知上、有意义地参与材料”，例如分析、综合、评价，而不是被动接收。

这种定义方向正确，但对教师有三个实践困难：

1. “让学生认知参与”“让学生真正思考”太抽象，不能直接告诉教师怎样设计活动。
2. 教师缺少标准来判断哪些 active learning 活动更好。
3. 教师缺少指南来改造已有作业，使它们更能提升学习。

ICAP 的贡献是把“认知参与”落到教师可观察的外显行为上。外显行为不完美，但它给教师和研究者一个可操作代理指标：学生在学习材料上到底做了什么。

### 2. 学习任务与参与模式是正交的

论文强调，instructional task 与 engagement mode 不是一回事。读文章、听讲座、看视频都可以以不同模式完成。

以阅读为例：

- Passive：安静地读，不做其他与学习相关的事。
- Active：划线、高亮、复制删除式总结。
- Constructive：自我解释、整合跨文本信息、用自己的话记笔记。
- Interactive：与伙伴互相提问和回答理解问题。

以听讲座为例：

- Passive：只听。
- Active：重复、背诵、照抄解题步骤、逐字记笔记。
- Constructive：出声反思、画概念图、提问。
- Interactive：在两人或小组中辩护、争论一个立场。

以看视频为例：

- Passive：只观看。
- Active：暂停、播放、快进、倒带。
- Constructive：解释视频里的概念，把视频内容与先前知识或其他材料比较。
- Interactive：与同伴讨论理由、相似点和差异。

这个“任务与模式正交”的判断很关键：不能说“做笔记一定有效”或“看视频一定被动”。同一个活动如果实现方式不同，会落在不同 ICAP 模式上。

### 3. Passive：接收信息，但没有额外外显学习行为

Passive mode 是学习者朝向教学材料并接收信息，但没有做其他与学习相关的外显行为。典型例子是只听讲座、不记笔记；只看视频、不暂停；只读文字、不划线、不解释。

论文承认，学生外显上看起来被动时，内部可能正在深度加工。这是 ICAP 的一个边界：ICAP 以外显行为作为代理指标，不等同于直接读出大脑内部过程。

在很多学习系统研究中，“是否 engaged”被粗略地当作是否注意材料、是否走神。ICAP 把这种“看着材料、没有走神”的状态放在最低的 passive 模式中，而不是把它等同于高质量认知参与。

### 4. Active：对材料做操作，但不生成新信息

Active mode 要求某种外显动作或物理/运动操作，例如：

- 旋转物体以检查部分；
- 指向、手势、操作材料；
- 暂停和倒带视频；
- 下划线、高亮；
- 复制解题步骤；
- 在菜单中选择理由；
- 在实验中混合化学物质。

Active 的判据不是“动了手”本身，而是这种动作是否让注意力聚焦到材料的特定部分。论文特别区分了 mindless motor behavior 与 focused manipulation：如果学生只是把全文大声读完，没有把注意力聚焦到具体句子或段落，这更接近 passive；如果学生在某些关键处加重语气，可能涉及 focused attention，就可能更接近 active。这里存在分类歧义。

Active 比 Passive 深，因为它通过操作和强调材料，可能激活相关先验知识，并把新信息整合进已有 schema。

### 5. Constructive：生成材料之外的新信息

Constructive mode 的核心判据是：学习者生成或外化出材料中没有的新信息。它不是简单重复、复制、选择或标记，而是产生额外想法。

典型例子包括：

- self-explaining：说出一句话或一个解题步骤对自己意味着什么；
- 生成文本中没有明说的推论；
- 为解题步骤提供理由；
- 画材料中没有提供的自由体图；
- 画概念图；
- 用自己的话记笔记；
- 提问、造题；
- 比较和对比案例；
- 整合两篇文本、文本和图表、多媒体资源；
- 制定计划；
- 归纳假设和因果关系；
- 类比；
- 预测；
- 反思和监控自己的理解；
- 建构历史现象的时间线。

关键区分是：

- 如果学生自我解释时只是逐字复述原文，那只是 active/repeating，不是 constructive。
- 如果物理题原解里没有图，学生自己画自由体图，这是 constructive。
- 如果原解里已有图，学生照抄图，这只是 active。

论文也澄清：ICAP 中的 constructive 比教育哲学中的 constructivism 更具体。它不直接等于“发现式学习”或“反直接教学”，而是从学习者视角定义为：生成超出所给材料的信息。教师脚手架可以和 constructive 一致，因为脚手架常常促使学生用“what else”“hint”“fill-in-the-blank”等方式生成回答，而不只是说 yes/no。

### 6. Interactive：不是一起做事，而是共同生成的对话

Interactive mode 在论文中被严格定义为满足两个条件的对话：

1. 双方话语主要是 constructive 的，即每个人都在生成超出材料和对方前一句的新信息。
2. 有足够频繁的 turn taking，即对话不是轮流小讲座，而是持续互相插入、追问、澄清、修正。

因此，ICAP 中的 interactive 不是泛泛的人机交互，也不是“两个人一起操作”。两个人互相抄作业只是 active 互动，不是 constructive，更不是真正 interactive。

真正 interactive 的例子包括：

- 互相捍卫和争论立场；
- 请求对方给出理由；
- 互相提问和回答；
- 彼此解释；
- 澄清、搭建、纠正对方贡献；
- 共同生成任何一方单独无法生成的新想法。

论文还区分 joint dialogue 与 individual dialogue。若一个人主导输出，另一个人只说 “uh-huh”“mmm” 或简单同意，主导者可能在 constructive 地学习，倾听者只是 active 地确认。这可以解释某些 collaborative learning 研究为什么看不到优势：小组合作不自动等于 interactive。

对计算机系统也一样：一个系统要求学生点击、选择、输入，不代表它在 ICAP 意义上 interactive。只有当系统诱发学习者生成性回应，并能给出内容相关的回应，使学习者在对话中继续建构，才更接近 interactive。

### 7. 四种知识变化过程：Store、Integrate、Infer、Co-Infer

论文把学习变化的粒度限定在 knowledge level，因为 ICAP 关心的是学生对复杂学校知识的个体理解如何提升。

它提出四种 broad knowledge-change processes：

| ICAP 模式 | 外显特征 | 知识变化过程 | 知识结果 | 认知结果 |
|---|---|---|---|---|
| Passive | receiving | Store | 新信息孤立存储 | recall |
| Active | manipulating | Integrate | 新信息与先验 schema 整合、补全 | apply |
| Constructive | generating | Infer | 整合后进一步推理、修订、重组 | transfer |
| Interactive | dialoguing | Co-Infer | 双方推理互相输入、螺旋累积 | co-create |

#### Passive / Store

Passive 下，新知识以相对孤立、封装的方式存储，未和先验知识充分整合。论文借用 inert knowledge：这种知识能在相同线索或相同上下文下被提取，但很难迁移。

论文举例：一个学生读过血小板和结痂知识，能复述“platelets made a shield on your cut”，但当被要求用自己的理论解释伤口愈合时，又说“被切断的血管死了，心脏停止给它送血”。这说明新材料被存了下来，但没有和旧理论整合。

#### Active / Integrate

Active 的操作会强调材料某些部分，从而激活相关先验知识或 schema。新信息随后被同化进被激活的 schema，填补空缺，使 schema 更完整、更稳固。

这类学习可以支持在相似情境中应用知识，例如解决类似问题、解释类似概念。论文把它称为至少达到 shallow understanding。

#### Constructive / Infer

Constructive 涉及 inferring，包括归纳、演绎、溯因，也包括修订、修复、重组、反思。学习者不仅补全 schema，还可能让 schema 更丰富、更连贯，甚至发生 accommodation。

论文例子是学生对循环系统从 “single-loop” 模型改为 “double-loop” 模型。constructive 活动如 self-explaining 可能促成这种知识结构变化。

Constructive 对概念领域尤其重要，因为它支持解释泛化、类比推理；对程序领域也重要，因为为步骤提供理由和根据能帮助程序迁移到新情境。

#### Interactive / Co-Infer

Interactive 的核心是 co-inferring：每个人都基于材料和自己的先验知识生成推论，随后把推论给对方，对方再基于材料、自己的 schema 和对方推论生成新的推论。这个过程形成累积、螺旋式共同建构。

论文的 Alice-Bob 假设例子说明：Alice 基于 Knowledge 1 和 Schema A 生成 Inference 1；Bob 基于 Knowledge 1、Inference 1 和 Schema B 生成 Inference 2；Alice 再把 Inference 2 纳入自己的推理。这样每一轮都可能把对方的推论变成自己的新输入。

### 8. ICAP 假设：更高模式包含更低模式，但学习结果不是无条件保证

论文认为，高层模式通常 subsume 低层模式：

- Active 包含接收材料，但多了操作和注意聚焦。
- Constructive 包含接收和操作，但多了生成新信息。
- Interactive 包含双方各自 constructive，并通过轮替对话互相输入。

由此推导出学习结果排序：

- Passive：minimal understanding，主要可回忆。
- Active：shallow understanding，可应用到相近情境。
- Constructive：deep understanding，更可能迁移。
- Interactive：deepest understanding，可能共同创造新理解。

但这个排序是关于模式之间的相对预测，不是对任何具体任务的无条件保证。论文后面明确列出会覆盖或违反 ICAP 预测的条件。

### 9. 实证验证：从四模式实验到具体活动比较

论文第二部分用多类证据验证 ICAP 假设。

#### 四模式实验

Menekse et al. (2013) 的材料科学学习实验包含四种条件：

- Passive：读文本。
- Active：读文本并高亮重要句子。
- Constructive：解释一张表达文本信息的图，但不读文本。
- Interactive：与同伴共同解释同一张图，也不读文本。

结果支持 ICAP，学习从一个模式到下一个模式大约提升 8%–10%。

#### 三模式研究

论文还寻找了文献中能被 ICAP 重解释为三种模式对比的研究，例如 Coleman、Brown & Rivkin (1997) 等，用来支持模式之间的相对排序。

#### 三类具体学习活动

论文重点检查三种活动在不同 ICAP 模式下的效果：

1. **Note taking**：做笔记不是天然有效。逐字记笔记更接近 active；用自己的话、回答 who-what 问题、生成性总结更接近 constructive；合作笔记是否更好取决于对话和生成质量。
2. **Concept mapping**：复制概念图、纠正概念图、自己生成概念图、合作生成概念图分别可能落到不同模式。关键不是“概念图”这个名词，而是学生是否生成新节点、新关系、新解释。
3. **Self-explaining**：self-explaining 常常是 constructive，因为它要求学习者解释一句话、一个概念或一个步骤对自己意味着什么；但如果只是从菜单选择解释，它更接近 active；如果与伙伴共同解释、回应对方贡献，它可成为 interactive。

#### Classroom studies

课堂研究也能被 ICAP 重解释：

- partial scaffolded outline 记笔记优于完整讲稿，因为前者要求学生主动补充，后者更被动。
- 建构概念图优于全班讨论，因为全班讨论对多数沉默学生来说可能只是 passive。
- 同伴讨论因果关系优于观看教师指出因果关系。
- jigsaw、peer tutoring 等互动结构优于个人搜集信息或填 guided notes。
- 两人一起做题不必然是 interactive：若两人只是共同选择静态图片，仍可能只是 active；若两人共同画分子结构、生成表示，才更接近 interactive。

### 10. Caveats：哪些因素会覆盖 ICAP 假设

论文第三部分明确说，ICAP 只是从“学生做什么”这个方面预测学习结果，是一个 gross first cut。至少三类因素会覆盖或改变 ICAP 预测。

#### 10.1 测评太浅，看不出差异

如果测评题只要求回忆或非常容易，Passive 也可能足够答对，因此不同 ICAP 模式的优势不会显现。

论文例子包括：解释提示在容易猜的题上没有优势，但在 hard-to-guess 和 transfer 问题上有显著好处。Menekse et al. (2013) 中，简单选择题在四模式之间没有差异，困难项目上才出现差异。

因此，ICAP 不只要求设计更深学习活动，也要求设计能测出深理解和迁移的 assessment。

#### 10.2 领域不适合自我推理，或学习者缺少相关 schema

有些简单程序性规则是人为规定、任意性强、无法通过逻辑推出的。例如英语冠词 a/an/the/no article 的许多规则和例外。要求学习者解释为什么使用某个冠词，未必带来更好学习，因为没有可推导的深层理由。

另一个情况是主题太难，学习者缺少相关 schema，无法靠 self-explaining bootstrap 到正确深理解。论文以 emergence 概念为例：学生习惯顺序因果链，缺少理解微观行为如何集体解释宏观模式的图式，因此回答提示只能帮助他们学到微观细节，未必帮助理解跨层 emergent relation。

这与 [[wiki/concepts/Cognitive Load Theory]] 和 [[wiki/syntheses/ICAP 与知识类型的适用边界]] 的判断一致：Constructive / Interactive 需要足够先验结构，否则可能变成过载或错误生成。^[inferred]

#### 10.3 同一模式内部，不同任务仍可能有不同效果

ICAP 主要预测模式之间的差异，不保证同一模式内所有活动等效。两个 interactive 活动可能因为任务结构不同，诱导不同认知加工，导致学习效果不同。

论文举了水平抛射运动学习例子：两人一起 invent 方法与两人 compare-and-contrast 案例都可被归为 interactive，但 invent 条件效果更好。更细分析发现，compare-and-contrast 让学生更关注单特征陈述；invent 促使学生连接速度、高度、距离等多个特征，从而更接近任务目标。

所以 ICAP 是粗粒度参与模式分析，不替代具体任务结构分析。

### 11. ICAP 作为研究和教学设计工具

论文把 ICAP 的用途分成四类。

#### 11.1 解释文献中的矛盾结果

同一个活动名词可能对应不同 ICAP 模式。比如“summarizing / note taking”在一些研究中有效，在一些研究中无效。ICAP 的看法是：不要问“总结是否有效”，而要问：

1. 它如何实现？是 active selection，还是 constructive explanation？
2. 它和什么 control condition 比较？Passive、Active、Constructive 还是 Interactive？
3. 学生实际如何执行？是否真的生成新信息？

因此，看似矛盾的研究结果可能在 ICAP 视角下是系统性的。

#### 11.2 选择更严格的 control condition

如果一个 intervention A 比 intervention B 好，不能马上说 A 的特殊机制有效。必须先看 A 和 B 的 ICAP 模式是否不同。

例如，如果 A 是 constructive，B 是 active，那么结果只能说明 A 更认知参与，而不能证明 A 的具体提示形式本身优越。要检验某种 metacognitive prompt 的独特价值，应当和另一个同样能诱发 constructive response 的 prompt 比较。

#### 11.3 评价学生输出

教师设计任务时可能以为任务是 constructive，例如让学生画概念图。但学生实际输出可能只是使用教师提供的六个节点，没有添加任何新节点或关系。这种执行实际上可能只是 active。

所以 ICAP 可以发展成 output coding rubric：通过检查学生是否生成了材料外的新信息，判断 intended mode 与 enacted mode 是否一致。

#### 11.4 指导教学设计

ICAP 可以帮助教师把活动往更高参与模式改造：

- 原本只讲课：加入 guided notes，把 passive 改成 active。
- 原本只让学生跟着听：加入概念图任务，把活动改成 constructive。
- 原本只让学生分组：要求每个人提出理由、回应同伴、修正解释，把“形式上的小组”改成真正 interactive。

对计算机学习环境设计也一样：选择题容易实现，但常常只到 active；自由生成、解释、对话更难实现，但更可能带来 constructive / interactive 学习。

### 12. 与其他理论的关系

#### 12.1 与 constructivism

ICAP 与建构主义相似，都把学习责任更多放在学习者身上，而不是只看教师传递。但 ICAP 比 constructivism 更细：它用外显行为和学习结果区分 active、constructive、interactive。

ICAP 中的 “constructing” 不是让学生自己发现事实，而是学习者用生成性活动把给定事实与先验知识整合、推理和理解。比如“心脏有双循环”这个事实可以直接告诉学生；constructive 的作用是让学生理解双循环意味着什么，而不是必须自己发现这个事实。

#### 12.2 与 cognitive load theory

认知负荷理论关注如何降低外在负荷、管理内在负荷，让有限工作记忆能处理材料。ICAP 关注如何诱发更高层级的学习者活动。

两者有时看起来方向相反：认知负荷理论常说负荷越高任务越难，学习可能越差；ICAP 则预测从 passive 到 interactive，活动更费力，但学习可能更好。

论文的调和方式是：如果更高参与带来的是 germane cognitive load，例如生成自我解释，那么更高负荷可能伴随更好学习。也就是说，并非所有负荷都坏，关键是它是否服务于 schema construction。

#### 12.3 与 Bloom’s taxonomy

Bloom taxonomy 主要分类 learning objectives：remember、understand、apply、analyze、evaluate、create，并要求教学和测评与目标对齐。

ICAP 关注的是达到这些目标的 means，即学习者在学习材料上做什么。Bloom 更像目标和评估框架，ICAP 更像学习过程和活动设计框架。二者互补。

此外，Bloom 不区分个人内 cognition 和人际 interaction；ICAP 明确认为 co-construction through interaction 可能产生最高学习结果。

### 13. 结论：ICAP 是“带行为指标的认知参与理论”

论文结论把 ICAP 定义为一套 operationalized definitions of engagement activities。它的优势是能跨大量课堂和实验研究看出规律：随着活动从 passive 到 active、constructive、interactive，学生经历不同知识变化过程，学习通常增加。

但论文也保留两个理论 caveat：

1. 各模式对应的 knowledge-change processes 仍是假设性解释，论文没有直接验证这些过程确实发生。
2. 可能存在其他不依赖 representation 和 memory 的理论解释，但 ICAP 当前的解释依赖知识变化过程这条因果链。

因此，ICAP 最稳妥的用法不是把 `I>C>A>P` 当作口号，而是把它作为一个诊断链条：

> 外显行为 → 认知参与模式 → 知识变化过程 → 知识结构变化 → 可回忆 / 可应用 / 可迁移 / 可共同创造。

## Integration Decisions

当前 wiki 已有 [[wiki/sources/ICAP Learning Framework Source Guide]]，它保存的是一篇中文介绍文章层；这篇原论文 source guide 应作为更高置信度的 primary source 层，用于支持关于 ICAP 定义、四模式判据、知识变化过程、实证证据和理论 caveat 的声明。

[[wiki/syntheses/ICAP 与知识类型的适用边界]] 已经把 ICAP 放进知识类型、学习阶段、认知负荷和先验图式的边界判断中。当前原论文进一步支持其中几个关键判断：

- ICAP 是学习活动的参与深度刻度，不是知识类型分类。
- `I>C>A>P` 是模式之间的相对假设，不是无条件学习处方。
- Constructive / Interactive 的效果依赖领域是否有可推理结构，以及学习者是否有足够先验 schema。
- 测评必须足够深，才能看出高层认知参与的优势。

## Open Questions

- ICAP 的 knowledge-change processes 是否能被更直接地测量，而不只是由行为和结果反推？
- AI tutor 如何判断学生的 actual enacted mode，而不是只根据任务设计推断 intended mode？
- 对程序技能、判断知识、审美能力、工程能力等不同知识类型，ICAP 与刻意练习、案例复盘、专家反馈之间应该怎样组合？
- 在人机对话中，什么条件下 AI 才真正提供 ICAP 意义上的 interactive，而不是让学生 passive 地消费解释？
