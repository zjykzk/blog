# Understanding as Concept Cloud Source Guide

这页保留本轮 inline 材料的入库导览：它把“理解”解释为脑中关于对象的一团“云”，而不是一句词典式定义。

## Why It Matters

这份材料的价值在于给“理解”一个可操作的形状模型：横向看相似结构如何展开，纵向看抽象层级如何移动。

它能补强当前 wiki 里已有的 [[wiki/concepts/Concept]]、[[wiki/concepts/Knowledge as Network]] 和 [[wiki/topics/Learning Methodology]]：概念提供边界，知识网提供连接，而理解之云解释“我懂了”在脑内可能意味着什么形状变化。^[inferred]

## Key Claims

- 理解不是定义，而是脑中关于对象形成的一团“云”。
- 这团云有两个方向：横向相似度和纵向抽象度。
- 横向相似度指新信息进入时，脑中与之相像的结构被点亮，并由最匹配的结构承接新信息。
- 纵向抽象度指同一对象可以在具体细节、局部状态、长期趋势和普遍模式等不同层级上被观看。
- 给孩子讲清楚一件事，要求讲者从高抽象层降到孩子已有经验所在的低层，并找到孩子脑中已有的相似结构。
- 重新创造一个东西，是沿抽象层级和相似结构分布反复摸清云的形状。
- 读完一本书却几乎没有留下东西，可能说明相关概念云的形状没有明显改变。
