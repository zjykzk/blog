# Software Methodology by Pan Jianyu

这页收纳《软件方法》相关的阅读入口与当前已经稳定下来的判断。

## 当前抽出的主线

- 起点不是功能清单，而是目标组织的改进愿景
- 主收敛链是：愿景 -> 业务用例 -> 业务序列图 -> 系统用例 -> 用例规约 -> 分析设计 -> 实现与验证
- 原始骨架可以压成三根：价值、边界、责任
- 更严密的闭环需要显式补出：行为、反馈

## 为什么值得单列

《软件方法》不是单纯的 UML 画图教程。它真正重要的地方在于把软件项目从"功能收集"拉回到"业务改进 -> 边界收缩 -> 责任落实"这条链上。
