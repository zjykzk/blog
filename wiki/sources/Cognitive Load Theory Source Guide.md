# Cognitive Load Theory Source Guide

> Source: 用户粘贴文章《认知负荷理论：因为文具多，所以是差生》（conversation capture, 2026-05-09）

## What It Covers

This source introduces cognitive load theory as a learning and teaching model: learning fails when new information congests the narrow gate of working memory, not because long-term memory lacks capacity.

The article uses Peru's One Laptop per Child program as an opening case. It argues that hardware spending can improve computer-use skill without improving math, reading, or grade progression, because the learning bottleneck sits at the micro-level of cognitive processing rather than the macro-level of equipment access.

## Key Points

- Human long-term memory is treated as effectively high-capacity, while working memory is narrow and can only process a few interacting elements at once.
- Learning turns external information into durable long-term memory by processing it through working memory and organizing it into schemas.
- Cognitive load is the pressure imposed on working memory. The article distinguishes intrinsic load, extraneous load, and germane load, while noting that newer Sweller work questions whether germane load should remain a separate category.
- Intrinsic load comes from the task's element interactivity: how many pieces must be coordinated at once.
- Extraneous load comes from irrelevant demands: noise, poor explanation, decorative multimedia, distracting devices, emotional conflict, platform friction, or unnecessary materials.
- Schemas compress multiple elements into one usable cognitive unit. A learner who has the schema for “中华人民共和国” or “distributive law” experiences fewer working-memory elements than a novice facing the same surface material.
- “差生文具多” is used as a local metaphor: extra tools can create a false feeling of control while adding extraneous load rather than building the missing schemas.
- Effective instruction manages intrinsic load by sequencing components, preheating prerequisites, automating local parts, and integrating them later.
- Direct instruction and worked examples are presented as efficient for novices because they reduce search cost and let working memory focus on schema acquisition.
- Guidance should fade after schemas form; exploration becomes more useful when the learner has enough prior knowledge to handle the search space.
- Teaching support should depend on prior knowledge: low-prior-knowledge learners benefit from high support, while high-prior-knowledge learners benefit from lower support and more transfer-oriented variation.
- AI can harm learning when used as a doer that replaces the learner's schema-building work. It can help when constrained as a tutor that decomposes complexity, removes extraneous load, and guides the learner step by step.

## Integration Decisions

This source should connect to [[wiki/maps/Learning Map]] as a learning-cognition source rather than become a broad topic page immediately.

The most promotable concept is cognitive load theory itself: the idea that effective teaching must respect working-memory bandwidth and grow schemas in long-term memory. A future [[wiki/concepts/Cognitive Load Theory]] page could separate intrinsic load, extraneous load, schema formation, worked examples, and expertise reversal into stable definitions. ^[inferred]

The source strengthens [[wiki/topics/Learning Methodology]] by adding a different constraint language. The existing methodology emphasizes questions, structure, output, and knowledge networks; cognitive load theory adds the micro-mechanism: learners need a sequence that keeps working-memory load below overload while schemas form. ^[inferred]

The source also clarifies one line in [[wiki/concepts/Knowledge as Network]]: psychological dependency is not the same as knowledge dependency. Some prerequisite ordering exists because working memory needs schema scaffolding, not because knowledge has a global pyramid shape. ^[inferred]

For [[wiki/concepts/AI Learning Tutor Loop]], the article provides a guardrail: AI should not remove the learner's productive work. It should reduce extraneous load and calibrate intrinsic load, while preserving the learner's need to answer, fail, retrieve, explain, and repair understanding.

This source should not be promoted as evidence that computers or AI are inherently bad for learning. Its stronger claim is conditional: tools harm learning when they add extraneous load or replace schema-building effort; tools help when they act as scaffolded tutors. ^[ambiguous]
