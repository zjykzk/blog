# Application and Creation Distinction

The strongest distinction is not "doing versus making." Both [[wiki/concepts/Application]] and [[wiki/concepts/Creation]] can change the world. The difference is where the main transformation happens.

Application starts with an available abstraction, method, procedure, model, or tool. Its problem is contextual fit: how should this structure be interpreted, adapted, and tested inside a concrete situation?

Creation starts with pressure from an insufficient or missing form. Its problem is ordered difference: how can available materials be opened, recombined, and stabilized into a new form that can be used, judged, or continued?

## Short Distinction

```text
Application = existing abstraction + contextual fit -> situated effect
Creation = materials + recombination + validation -> new durable order
```

Application can still be intelligent, experimental, and non-mechanical. [[wiki/sources/How People Learn Transfer Source Guide]] supports this by treating transfer as flexible adaptation to new problems, not as memory replay.

Creation can still use existing knowledge and domain skill. [[wiki/sources/Amabile Componential Theory of Creativity Source Guide]] supports this by defining creative work as novel and appropriate, and by tying creativity to domain skills, creative processes, task motivation, and social environment.

## Why "Not Copying" Is Still Application

Weak application copies the surface form of a method. Strong application keeps the underlying structure but changes how it meets the case. In learning-theory terms, the learner must know when, where, why, and how a principle applies, not only remember the principle.

This is why [[wiki/concepts/Application]] belongs near transfer, reflective inquiry, and professional judgment. Dewey's reflective inquiry cycle begins with difficulty, defines the problem, entertains possible solutions, reasons through consequences, and tests by observation or experiment. That supports application as situated inquiry rather than procedure following. See [[wiki/sources/Dewey Reflective Inquiry Source Guide]].

## Why "New" Is Not Enough For Creation

Creation is not merely novelty. Amabile's componential theory requires novelty plus appropriateness to a goal. Csikszentmihalyi's systems model adds that creativity is also socially and historically situated: creative products are recognized inside domains and fields, not only inside the creator's private intention.

This supports the wiki's current phrase "difference entering order." A random variation can be new without being creative. A created thing must become coherent enough to survive use, interpretation, expert judgment, or continuation. See [[wiki/sources/Csikszentmihalyi Systems Model of Creativity Source Guide]].

## Design Theory Bridge

Design sits between application and creation. Simon's design theory frames design as changing existing situations into preferred ones; that can be mostly application when the preferred state and method are clear, and mostly creation when the problem framing, future state, and means must be invented together.

Schön's reflective-practice account adds that designers often work by conversing with the situation: they make a move, encounter surprise, reinterpret the situation, and invent a new move. That makes design a practical bridge: application becomes creative when fitting an abstraction forces a reframing of both the method and the situation. See [[wiki/sources/Schon Reflective Practice Source Guide]] and [[wiki/sources/Simon Sciences of the Artificial Source Guide]].

## Relation to Bloom's Taxonomy

The revised Bloom taxonomy places "apply" and "create" as different cognitive process categories. Apply is using or carrying out a procedure in a situation; create is generating, planning, or producing a coherent whole or new pattern. See [[wiki/sources/Revised Bloom Taxonomy Source Guide]].

The taxonomy is useful but should not be overread as a rigid ladder. In real work, application can contain analysis and evaluation, and creation depends on prior understanding and application. The distinction is functional, not a status hierarchy. ^[inferred]

## Working Test

Ask what would count as success:

- If success means "the method worked in this situation," the center is application.
- If success means "a new form now exists and can be used, judged, or continued," the center is creation.
- If success means "the method, situation, and standard all changed through action," the work is crossing from application into creation.
