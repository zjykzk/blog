# Learning Methodology Across Sources

把 wiki 里散落在不同笔记中的学习相关结论，按 [[wiki/topics/Learning Methodology]] 的七条原则做一次对齐。目的：看清楚哪些原则已经被多个独立来源印证、哪些还只是一家之言、哪些源之间有冲突需要再想。

## 每条原则的跨源证据与差异

### 1. 从问题进

- 《认知天性》的"检索式学习"本质是：学之后要能被问题调出来才有效——所以学之前最好就带着问题，这样存和取的路径一致。
- 《得到高效学习方法》明说"为解决问题"。
- 天才案例最硬：几乎没有例外。
- **孩子学习**这条没明说——因为孩子的学习更多是课表驱动的。这是个现实张力：七条原则是给**主动学习者**的，不完全适用于义务教育阶段。

### 2. just-in-time

- 只有天才案例支持，几本教科书化的学习法都不这么讲。
- 可能原因：主流学习理论服务于学校体系，学校必须 just-in-case 地打基础。
- 对成人、对自学者，这条是解放性的；对学生，要小心用。

### 3. 第一性原理还原

- 《认知天性》"耗费心血的学习"是一种温和版本：逼自己努力的那一刻，认知加工深，记忆牢。
- **孩子学习**"自己推导公理定理""做例题自己推一遍"是直接对齐。
- 《思维框架》列第一性原理为独立思维模型。
- 多源独立印证，是最硬的一条。

### 4. 抓结构层

- 《得到高效学习方法》结构主义和自然主义的二分，几乎就是这条原则的早期表述。
- **孩子学习**"题目是例题变形和组合"是低阶版本——变形和组合就是结构化看题。
- 《思维框架》的"地图&模型思维"直接对应结构层。
- 天才跨域迁移的全部证据。

### 5. 输出即学习

- 《认知天性》"检索式学习"是单向输出（考自己）。
- 《得到高效学习方法》"写摘要、录音、录像"是完整输出。
- 费曼技巧是对外输出。
- 这条原则的梯度：**对自己讲 → 写下来 → 对空气讲 → 对真人讲 → 教一个听不懂的人**。梯度越往后，压力测试越强。

### 6. 给网织密

- 《认知天性》交错练习、间隔重复，是给网织密的具体手法。
- **孩子学习**"全学科阅读"是在横向织网。
- 天才低产期是宏观时间尺度的织网。
- 这条在主流学习法里被低估——它反直觉，因为"看起来没产出"。但**爆发是网密到临界的自然结果**，不是投入更多新内容的结果。

### 7. 找真懂的活人

- 《得到高效学习方法》直接列"找老师"。
- **孩子学习**强调"专注老师，不是提前学"——信老师这个活人。
- 天才几乎都有一个关键人物点化过。
- 这条在在线教育时代被严重低估：MOOC 和书看起来高效，但**元信息**（这门学问长什么样）只从活人那里来。

## 源间冲突

一个真正的冲突：**孩子学习**里"专注课内、专注课本、专注老师"，和**天才路径**里"just-in-time、跳着学、自己选路径"——看起来方向相反。

怎么统一？——**阶段不同**：

- 早期（打地基阶段）：need 高度结构化的课内、老师、课本。基本功不能 just-in-time。
- 中后期（主动学习阶段）：按七条原则走。
- **"基本功"的边界**是什么：是真硬依赖（第一性原理必备的推理训练、语言的词汇量、基础记忆容量）。这些必须 just-in-case。

这个冲突的解是 [[wiki/concepts/Knowledge as Network]] 里的"硬依赖 vs 脚手架"：硬依赖必须按顺序打，脚手架可以跳。
