# 大语言模型时代知识图谱的角色转变：从结构化知识表示到 GraphRAG 增强生成

## 摘要

大语言模型的发展改变了知识获取、知识表达和知识生成的方式。与传统知识图谱相比，大语言模型具有更强的自然语言理解和文本生成能力，但也存在幻觉、知识更新滞后、可解释性不足和领域可靠性有限等问题。知识图谱以实体、关系和语义约束为核心，能够提供结构化、可追踪和可校验的知识组织方式。本文围绕知识图谱与大语言模型的协同关系展开综述，首先比较知识图谱的显式知识表示与大语言模型的隐式参数化知识表示，然后分析知识图谱增强大语言模型的主要机制，包括检索增强、事实约束、语义校验和可解释推理；进一步讨论大语言模型在知识图谱构建、补全和更新中的辅助作用。在此基础上，本文重点分析 GraphRAG 的技术逻辑及其意义，指出 GraphRAG 将检索增强生成从文本片段相似性推进到实体关系结构检索。研究认为，知识图谱并未被大语言模型取代，而是在生成式人工智能时代转向可信生成、结构化检索和知识治理的基础设施。未来研究需要进一步解决图谱质量、动态更新、冲突裁决和评估标准等问题。

**关键词**：知识图谱；大语言模型；检索增强生成；GraphRAG；可信人工智能；知识治理

## 1 引言

知识图谱是人工智能和语义技术中的重要知识表示方式。它通过实体、关系和属性描述现实世界中的对象及其关联，使知识能够以图结构形式被存储、查询和推理。自知识图谱概念受到广泛关注以来，其已被应用于搜索引擎、智能问答、推荐系统、金融风控、医疗知识管理和教育知识建模等场景。与普通文本不同，知识图谱强调知识的结构化表达和语义关联，因此适合处理实体关系明确、推理路径清晰、知识来源需要追踪的任务。

近年来，大语言模型迅速发展，改变了人们对知识系统的理解。以 GPT 系列、PaLM、LLaMA 等为代表的大语言模型，能够基于大规模语料训练获得强大的语言理解、文本生成、代码生成和复杂任务处理能力。与传统知识图谱不同，LLM 并不主要依赖显式的实体—关系结构，而是通过模型参数吸收语料中的语言模式和事实关联。用户可以直接用自然语言提问，模型则以自然语言生成回答。这种交互方式降低了知识获取门槛，也使许多原本依赖知识库、规则系统或专门模型的任务可以通过统一的语言模型完成。

但 LLM 的发展并没有消除知识组织问题。相反，它使知识可信性问题更加突出。大语言模型可能生成形式流畅但事实错误的内容，即通常所说的“幻觉”。模型内部知识还受到训练数据时间范围限制，难以及时反映最新事实。对于专业领域问题，通用 LLM 可能缺乏足够可靠的领域知识。此外，LLM 的参数化知识表示缺乏透明性，用户难以判断答案依据何种来源、经过何种推理过程生成。对于教育、医疗、金融、科研和企业知识管理等场景，答案的可追踪性、可解释性和可验证性往往与答案本身同样重要。

在这种背景下，知识图谱的价值需要重新审视。一个常见判断是，大语言模型已经能够生成大量知识性回答，因此传统知识图谱的重要性下降。本文认为，这种判断忽视了 KG 与 LLM 在知识表示方式上的根本差异。LLM 擅长自然语言理解和生成，但其知识主要隐含在模型参数中；KG 擅长实体关系建模和语义约束，其知识以显式结构存在。二者不是简单替代关系，而是互补关系。知识图谱可以为 LLM 提供外部事实、结构化语义和可解释路径；LLM 则可以降低知识图谱构建、更新和自然语言交互的成本。

检索增强生成（Retrieval-Augmented Generation, RAG）为这种互补关系提供了技术基础。RAG 的基本思想是在模型生成前先检索外部知识，再将检索结果输入模型，从而减少模型对参数记忆的依赖。传统 RAG 多以文本片段为检索单位，通过向量相似度寻找相关内容。这种方法在开放域问答和文档问答中取得较好效果，但对多跳推理、跨文档知识整合和复杂实体关系表达仍有不足。GraphRAG 正是在这一基础上发展起来的方向。它将知识图谱或图结构引入 RAG，使检索对象从文本块扩展为实体、关系、路径、社区和子图。

GraphRAG 的出现说明，知识图谱在 LLM 时代并没有退出知识系统，反而以新的形式进入生成式 AI 的基础架构。它不再只是传统意义上的知识库，也不只是语义搜索工具，而是成为组织外部知识、约束生成内容、支持证据追踪和提升系统可信度的重要机制。通过 GraphRAG，知识组织方式从“相似文本检索”走向“实体关系结构检索”，这为企业知识库、科研知识管理和教育知识图谱等场景提供了新的技术路径。

本文围绕“大语言模型时代知识图谱的角色转变”展开综述，重点讨论知识图谱与 LLM 的协同机制及 GraphRAG 的技术意义。全文结构如下：第二章比较知识图谱与大语言模型的知识表示差异；第三章分析知识图谱增强 LLM 的主要机制；第四章讨论 LLM 辅助知识图谱构建的方式和风险；第五章重点分析 GraphRAG 的技术逻辑、优势和局限；第六章讨论企业知识库、科研知识管理和教育知识图谱三个应用场景；第七章总结当前挑战与未来方向；第八章给出结论。

## 2 知识图谱与大语言模型的知识表示差异

知识图谱是一种以实体、关系和属性为核心的结构化知识表示方式。其基本表达形式通常是“实体—关系—实体”三元组，例如“作者—撰写—论文”“疾病—具有症状—发热”。这种表示方式将现实世界中的对象、概念及其关联转化为可计算、可查询和可推理的结构。Hogan 等人将知识图谱视为一种面向语义表达、数据互联和知识推理的图结构知识系统[1]。Ji 等人的综述进一步将知识图谱研究划分为知识表示、知识获取、知识融合、知识推理和知识应用等方向[2]。由此可见，知识图谱并不只是数据库中的一种存储形式，而是一种组织知识、表达语义关系和支持机器推理的基础设施。

知识图谱的优势在于显式性。实体和关系被清楚地表示在图结构中，用户可以追踪某个结论来自哪些实体、关系或路径。例如，在医学知识图谱中，某种疾病、症状、药物和禁忌症之间的关系可以被明确建模；在教育知识图谱中，知识点、课程、题目和学习路径之间的依赖关系可以被结构化呈现。这种显式结构使知识图谱适合用于语义搜索、智能问答、推荐系统和可解释推理。与普通文本相比，知识图谱能够减少自然语言表达中的歧义，使机器更容易识别概念之间的逻辑关系。

但知识图谱也有明显限制。第一，图谱构建成本高。高质量知识图谱通常需要领域专家参与本体设计、实体规范、关系定义和数据校验。第二，图谱更新较慢。现实知识不断变化，图谱中的实体和关系也需要动态维护。第三，图谱覆盖范围有限。即使是大型知识图谱，也难以覆盖所有领域知识，尤其是长尾知识、新兴概念和隐性经验。第四，图谱推理能力依赖图结构质量。如果实体抽取错误、关系定义不清或数据来源不可靠，后续推理也会受到影响。

与知识图谱不同，大语言模型主要通过参数化方式存储和调用知识。LLM 在大规模语料上训练，通过模型参数吸收语言模式、事实关联和语义结构。用户提出问题时，模型并不是直接查询一个显式知识库，而是根据输入上下文和参数中形成的统计关联生成回答。这种机制使 LLM 具有很强的语言理解、文本生成、概括和迁移能力。它可以在没有专门建模实体关系的情况下完成问答、摘要、翻译、代码生成和推理任务。

然而，LLM 的知识表示具有隐式性。模型知道什么、不知道什么，以及为什么给出某个答案，通常并不透明。Bender 等人对大规模语言模型的风险提出批判，指出模型可能生成形式流畅但缺乏真实理解的文本[5]。幻觉问题也成为自然语言生成研究中的重要议题。Ji 等人的综述指出，生成模型可能产生与事实不一致、与输入不一致或无法由来源支持的内容[6]。在课程论文语境下，可以将这一问题概括为：LLM 善于生成语言，但不总是可靠地表达事实。

因此，知识图谱与大语言模型的关系不宜理解为替代关系。知识图谱代表显式知识结构，强调实体、关系、约束和可追踪性；大语言模型代表隐式语言能力，强调语义理解、自然表达和上下文生成。前者适合组织和约束知识，后者适合理解和生成语言。两者的结合，正是当前知识增强生成、可信问答和 GraphRAG 研究的重要基础。

## 3 知识图谱增强大语言模型的主要机制

大语言模型的局限推动了外部知识增强方法的发展。由于 LLM 的知识主要存储在参数中，其回答可能受到训练语料时效性、覆盖范围和模型生成机制的影响。当问题涉及最新事实、专业领域知识或多跳关系推理时，仅依赖参数记忆往往不够可靠。Kandpal 等人的研究表明，大语言模型在长尾知识上表现有限[7]。Mallen 等人也讨论了参数化记忆与非参数化记忆的差异，指出在某些场景下，外部检索知识比模型内部记忆更可靠[8]。这为知识图谱增强 LLM 提供了现实依据。

第一种机制是检索增强。知识图谱可以作为外部知识源，为 LLM 提供相关实体、关系、路径或子图。用户提出问题后，系统先从图谱中检索相关知识，再将检索结果作为上下文输入 LLM。与普通文本检索相比，基于知识图谱的检索不仅能找到相似内容，还能利用实体关系发现结构性关联。例如，若用户询问“某一算法与哪些任务和数据集相关”，知识图谱可以沿着“算法—应用任务—评测数据集—代表论文”的路径组织答案。这样，LLM 生成的内容不再完全依赖参数记忆，而是受到外部结构化知识的支持。

第二种机制是事实约束与语义校验。LLM 生成答案后，可以利用知识图谱检查其中的实体关系是否合理。例如，如果模型生成“某药物可以治疗某疾病”，系统可以查询医学知识图谱，判断该关系是否存在、是否有禁忌关系、是否需要限定条件。知识图谱在这里承担“事实边界”的作用。它不能保证所有答案都正确，但可以减少明显违背已有知识结构的生成内容。对于医疗、金融、法律、教育等高可靠场景，这种约束尤其重要。

第三种机制是可解释推理。LLM 的回答往往难以解释，因为模型内部参数无法直接转换为人类可读的推理路径。知识图谱则可以通过路径、子图和关系链提供解释依据。例如，在推荐系统中，系统不仅可以告诉用户“推荐某本书”，还可以说明推荐理由是“该书与用户已读书籍共享主题、作者或知识领域”。在问答系统中，知识图谱可以展示答案涉及的实体和关系路径，使用户看到答案生成背后的知识依据。这种解释不一定等同于模型内部真实推理过程，但能为用户提供可检查的证据结构。

第四种机制是领域适配。许多专业场景中，通用 LLM 缺乏足够准确的领域知识。通过构建领域知识图谱，可以将特定行业、课程、企业或科研领域中的概念体系输入模型，使模型生成更符合领域语境的回答。例如，企业知识库可以把产品、客户、流程、制度和案例组织成图谱；教育知识图谱可以把知识点、课程目标、题目和学习路径关联起来；科研知识图谱可以把论文、作者、方法、任务和数据集连接起来。这些图谱为 LLM 提供了比通用语料更精确的领域语义结构。

因此，知识图谱增强 LLM 的核心价值不只是“提供更多知识”。如果只是补充文本资料，普通检索系统也能完成一部分任务。知识图谱更重要的作用在于提供可检查、可追踪、可约束的知识结构。它使 LLM 的生成过程能够与外部事实、实体关系和语义路径发生连接，从而缓解幻觉、不可解释和领域可靠性不足等问题。

## 4 大语言模型辅助知识图谱构建

知识图谱不仅可以增强 LLM，LLM 也可以反过来辅助知识图谱构建。传统知识图谱构建通常包括数据采集、实体识别、关系抽取、实体对齐、本体设计、图谱融合和质量校验等环节。这些步骤需要大量标注数据、规则设计和人工审核。随着 LLM 的发展，许多原本依赖专门模型或人工规则的任务，可以通过提示工程、少样本学习或指令微调得到辅助。

首先，LLM 可以用于实体识别和关系抽取。面对大量非结构化文本，LLM 可以根据提示从段落中识别人物、机构、地点、概念、事件、方法、数据集等实体，并抽取它们之间的关系。例如，在科研文献处理中，LLM 可以识别“方法—应用任务”“论文—使用数据集”“作者—所属机构”等关系；在企业文档处理中，LLM 可以抽取“产品—功能”“流程—责任部门”“问题—解决方案”等关系。这降低了知识图谱冷启动阶段的成本。

其次，LLM 可以辅助图谱补全和知识更新。已有知识图谱往往存在实体缺失、关系缺失和属性不完整等问题。LLM 可以根据上下文推测潜在关系，或根据新文档发现新增实体和新关系。例如，当一篇新论文提出某种方法并应用于某个任务时，LLM 可以从摘要或正文中抽取相关信息，并建议将其加入科研知识图谱。知识图谱由此获得更强的动态更新能力。

再次，LLM 可以辅助本体生成和领域概念建模。本体设计是知识图谱构建中的难点，因为它要求明确概念层级、关系类型和约束规则。LLM 可以根据领域文本归纳术语体系，生成初步概念分类，提出可能的实体类型和关系类型。例如，在教育领域，LLM 可以根据课程大纲整理“知识点—先修知识—学习目标—题型”的结构；在企业知识管理中，LLM 可以根据制度文件归纳部门、流程、角色和职责之间的关系。

但 LLM 辅助知识图谱构建也带来风险。最主要的问题是幻觉污染。LLM 可能生成看似合理但并不存在的实体关系。如果这些结果未经校验就写入图谱，知识图谱本身会变成错误知识的载体。这样一来，后续基于图谱的检索、推理和生成都会受到影响。因此，LLM 可以降低知识图谱构建门槛，但不能替代验证机制。更合理的流程是：LLM 负责候选实体和候选关系生成，规则系统、外部数据库、人工专家或多源证据负责校验。

这说明 KG 与 LLM 的协同是双向的。一方面，KG 为 LLM 提供结构化知识约束；另一方面，LLM 为 KG 提供自动化构建和更新能力。二者结合的关键，不是让其中一方完全替代另一方，而是在生成能力与结构约束之间建立可靠的工作流。

## 5 GraphRAG：知识图谱与 RAG 的融合趋势

RAG，即检索增强生成，是连接外部知识与语言模型的重要方法。Lewis 等人提出的 Retrieval-Augmented Generation 为知识密集型自然语言处理任务提供了经典框架[9]。其基本思想是：模型在生成答案之前，先从外部语料库中检索相关文档或文本片段，再基于检索结果生成回答。与完全依赖参数记忆的语言模型相比，RAG 可以引入更新、更具体、更可追溯的外部信息。

传统 RAG 的主要检索单位通常是文本块。系统先将文档切分成若干片段，再通过向量表示计算用户问题与文本片段之间的语义相似度。这种方法简单有效，适合处理大量非结构化文档。但它也有局限。第一，文本切块可能破坏原文结构，使实体关系被割裂。第二，向量相似度更擅长找到语义相近内容，却不一定能发现多跳关系。第三，普通 RAG 对跨文档综合能力有限。当问题需要整合多个实体、多个来源和多层关系时，单纯检索相似文本片段可能不足。

GraphRAG 正是在这一背景下受到关注。GraphRAG 将图结构引入检索增强生成过程，使系统不只检索文本片段，还能检索实体、关系、路径、社区和子图。Edge 等人提出的 Graph RAG 方法将图结构用于查询聚焦摘要，体现了从局部文本检索到全局图结构组织的转变[14]。在这种框架中，系统通常先从文档中抽取实体和关系，构建知识图谱或语义网络；随后根据用户问题定位相关实体和子图；最后将图谱中的结构化信息、社区摘要或关系路径输入 LLM，用于生成答案。

GraphRAG 的第一项优势是支持多跳推理。许多复杂问题并不能通过一个文本片段回答，而需要沿着多个实体和关系进行推导。例如，用户可能询问“某个研究方向受到哪些方法影响，又应用在哪些任务中”。普通 RAG 可能检索到若干相关段落，但不一定能清楚组织这些段落之间的关系。GraphRAG 可以通过图结构连接“研究方向—方法—任务—论文—数据集”等节点，使答案更具结构性。

第二项优势是支持跨文档知识整合。企业文档、科研论文和政策材料往往分散在多个文件中。普通 RAG 容易把每个文本块视为独立片段，而 GraphRAG 可以把不同文档中的同一实体、相关概念和相似关系连接起来。这样，系统能够从局部文档问答转向全局知识网络问答。对于组织知识库和科研知识管理，这一点尤其重要。

第三项优势是提高答案的可解释性。GraphRAG 可以在生成答案时保留相关实体、关系和路径，使用户看到答案依赖哪些知识节点。相比仅给出若干文本来源，图结构能够更清楚地展示知识之间的关联。例如，在教育场景中，系统可以说明某个知识点为什么是另一个知识点的先修内容；在科研场景中，系统可以说明某种方法如何从早期模型演化而来。

不过，GraphRAG 并不是没有代价。它首先依赖图谱质量。如果实体抽取错误、关系识别不准或图谱更新滞后，GraphRAG 的生成质量也会下降。其次，GraphRAG 的工程复杂度高于普通 RAG。它不仅需要文档切分和向量索引，还需要实体识别、关系抽取、图存储、社区发现、路径检索和结果融合。再次，GraphRAG 的评估标准尚不成熟。普通 RAG 可以评估检索命中率、答案相关性和事实一致性；GraphRAG 还需要评估图结构质量、子图选择质量、关系路径合理性和生成解释质量。

因此，GraphRAG 的意义不只是“让 RAG 多使用一种数据结构”。更准确地说，它把知识组织方式从“文本片段相似性”推进到“实体关系结构”。这与知识图谱在 LLM 时代的角色转变相呼应：知识图谱不再只是独立的知识库或语义搜索工具，而是成为生成式 AI 系统中的知识组织层、检索结构层和事实约束层。通过 GraphRAG，KG 与 LLM 的关系从外部连接走向深度融合。

## 6 典型应用场景

知识图谱与大语言模型的结合，不只是模型能力改进问题，也关系到知识系统如何在真实场景中组织、检索和使用。与通用聊天机器人相比，KG + LLM 或 GraphRAG 更适合知识密度高、实体关系复杂、答案需要来源支撑的场景。企业知识库、科研知识管理和教育知识图谱，是三类较有代表性的应用方向。

### 6.1 企业知识库与智能问答

企业内部知识通常分散在制度文件、产品文档、会议纪要、客户案例、工单系统和业务流程中。这类知识具有三个特点：来源分散、更新频繁、上下文依赖强。传统关键词检索只能返回相关文档，用户还需要自己阅读、比较和归纳。普通 RAG 可以改善这一问题，将相关文本片段输入 LLM，由模型生成自然语言答案。但如果问题涉及多个部门、多个流程或多个产品之间的关系，仅依赖文本片段相似度仍然不够。

知识图谱可以把企业知识组织成实体和关系网络。例如，实体可以包括部门、岗位、产品、客户、流程、制度、风险类型和解决方案；关系可以包括“负责”“依赖”“适用”“触发”“解决”“引用”等。LLM 在回答问题时，可以先通过图谱定位相关实体，再沿着关系路径找到制度依据、处理流程和责任主体。这样，企业问答系统不仅能回答“是什么”，还能回答“由谁负责”“依据什么文件”“与哪些流程相关”。

GraphRAG 在企业知识库中尤其有价值。它可以将文档中的局部信息整合为全局知识网络，再根据用户问题提取相关子图。与普通 RAG 相比，这种方式更适合回答跨文档问题。例如，“某客户投诉涉及哪些产品问题、历史工单和内部处理流程”并不是单个文本片段能够完整回答的问题。GraphRAG 可以通过客户、产品、工单、流程和责任部门之间的关系组织答案，使生成内容更接近业务逻辑。

不过，企业场景也暴露出 KG + LLM 的落地难点。企业数据格式不统一，图谱构建前需要大量清洗和标准化；内部知识具有权限边界，系统必须控制不同用户可访问的实体和关系；企业知识经常变化，图谱和向量索引都需要持续更新；模型生成的答案必须标注来源，否则很难被业务人员信任。因此，企业知识库中的 GraphRAG 不应只追求回答流畅，而应重视来源追踪、权限控制和知识更新机制。

### 6.2 科研知识管理

科研知识具有明显的网络结构。论文、作者、机构、研究任务、方法、数据集、实验指标和结论之间存在复杂关系。传统文献检索通常以关键词、作者或引用关系为入口，适合查找单篇论文，却不擅长呈现某个研究方向的整体结构。随着论文数量快速增长，研究者需要的不只是“找到文献”，还包括理解研究脉络、识别方法演化、发现知识空白和比较不同研究路线。

知识图谱适合表达科研知识网络。例如，节点可以是论文、作者、机构、方法、任务、数据集、评价指标和研究结论；边可以表示“提出方法”“使用数据集”“改进模型”“解决任务”“引用论文”“属于领域”等关系。这样的科研知识图谱可以支持多种任务：根据一个方法追踪其来源和后续改进；根据一个任务找到常用数据集和评价指标；根据一个研究主题识别核心论文和代表性作者。

LLM 可以增强科研知识图谱的使用体验。用户可以用自然语言提问，例如“近年来知识图谱增强大语言模型的主要路线有哪些”“GraphRAG 与传统 RAG 的差异体现在哪些任务中”。系统先从科研图谱中检索相关实体、论文和关系，再由 LLM 生成结构化回答。这样，用户不必手动浏览大量论文，也能获得初步的研究地图。

GraphRAG 对科研知识管理的意义在于，它可以把文献检索从“相似论文推荐”推进到“研究关系推理”。普通向量检索可能找到语义相似的论文摘要，但不一定能解释这些论文之间的关系。GraphRAG 可以利用图结构回答更复杂的问题，例如“某个方法从哪些早期技术发展而来”“哪些论文共同使用了同一数据集但得出不同结论”“某个研究方向中尚未充分探索的交叉点在哪里”。这些问题依赖实体关系，而不只是文本相似度。

但科研知识图谱也面临准确性问题。论文中的方法、任务和结论往往表达复杂，LLM 自动抽取时容易遗漏限定条件。例如，一篇论文可能只在特定数据集、特定指标或特定实验设置下取得较好效果。如果抽取过程忽略这些条件，就可能把局部结论误写成一般结论。因此，科研知识图谱需要保留来源、上下文和证据强度。对课程论文而言，这一点可以作为 KG + LLM 应用中的重要反思：结构化知识并不天然等于可靠知识，结构化过程本身也需要质量控制。

### 6.3 教育知识图谱

教育领域是知识图谱较早受到关注的应用方向之一。课程、知识点、学习目标、题目、教材章节和学生能力之间存在清晰的层次关系与依赖关系。例如，学习“深度学习”通常需要先掌握“线性代数”“概率论”“机器学习基础”等知识；学习某个编程概念可能需要先理解变量、函数和控制结构。教育知识图谱可以把这些概念及其关系结构化表示，用于个性化学习、智能推荐、学习路径规划和自动问答。

在 LLM 时代，教育知识图谱的价值进一步增加。LLM 可以生成解释、例题、反馈和学习建议，但如果缺乏课程结构约束，生成内容可能过难、过浅，或与学习者当前水平不匹配。知识图谱可以为 LLM 提供课程结构和先修关系，使其回答更符合学习路径。例如，当学生询问某个概念时，系统可以先判断该概念依赖哪些先修知识，再生成适合当前阶段的解释。

GraphRAG 可以支持更复杂的教育问答。普通 RAG 可能根据教材片段回答问题，而 GraphRAG 可以同时考虑知识点关系、题目类型、错误模式和学习路径。例如，学生做错一道题后，系统可以沿着“题目—考查知识点—先修知识—常见错误—推荐练习”的路径生成反馈。这种反馈比单纯解释题目答案更有教学价值，因为它关注的是学生知识结构中的缺口。

教育场景也需要谨慎处理生成内容。LLM 可能生成看似合理但不符合课程标准的解释，也可能给出过度简化的学习建议。知识图谱可以提供约束，但教育决策仍应考虑学习者差异、教学目标和教师判断。因此，KG + LLM 在教育中的合理定位，不是替代教师，而是辅助知识组织、诊断和反馈。

## 7 挑战与未来方向

知识图谱、LLM 与 GraphRAG 的结合展示了较强潜力，但当前仍面临多方面挑战。这些挑战既包括技术问题，也包括知识治理、系统评估和应用落地问题。

### 7.1 图谱构建与维护成本

高质量知识图谱的构建成本仍然较高。即使 LLM 可以辅助实体抽取、关系识别和本体生成，图谱仍需要领域专家校验。不同领域对实体类型、关系语义和约束规则的要求不同，无法完全依靠通用模型自动完成。特别是在医疗、法律、金融和教育等领域，错误关系可能带来严重后果。

维护成本同样重要。知识不是静态的，企业制度会更新，科研文献会增加，课程内容会调整，政策和标准也会变化。如果图谱无法及时更新，GraphRAG 检索到的结构化知识就可能过时。未来研究需要关注动态知识图谱、增量更新和版本管理，使 KG 能够适应持续变化的知识环境。

### 7.2 LLM 生成内容对图谱质量的影响

LLM 可以降低图谱构建门槛，但它也可能把错误信息引入图谱。由于 LLM 具有生成式特征，它可能补出不存在的关系、混淆相似实体，或忽略事实限定条件。如果这些生成结果未经校验就进入知识图谱，图谱会成为错误知识的放大器。

因此，未来 KG 构建流程应采用“生成—验证—入库”的模式。LLM 可以负责提出候选实体和关系，但入库前需要经过来源核验、规则检查、置信度评估或人工审核。对于高风险领域，还应记录每条关系的来源、时间、证据等级和适用范围。这样，知识图谱才能成为可信知识基础，而不是生成式内容的简单堆积。

### 7.3 KG 与 LLM 冲突时的裁决机制

当 LLM 的生成结果与知识图谱中的事实冲突时，系统应如何处理，是一个关键问题。简单地让 KG 覆盖 LLM 并不总是合理，因为图谱也可能过时或不完整；完全相信 LLM 也不可靠，因为模型可能产生幻觉。更合理的方式是建立冲突检测和裁决机制。

裁决机制至少需要考虑四类信息：第一，知识图谱中相关事实的来源和更新时间；第二，LLM 生成内容的外部证据；第三，不同来源之间的一致性；第四，应用场景的风险等级。在低风险场景中，系统可以提示存在不确定性；在高风险场景中，系统应拒绝给出确定答案，并要求人工确认。这种机制体现出 KG + LLM 系统从“生成答案”走向“管理知识可信度”的转变。

### 7.4 GraphRAG 的评估标准

GraphRAG 的评估比普通 RAG 更复杂。普通 RAG 通常关注检索相关性、答案准确性和事实一致性。GraphRAG 除了这些指标，还需要评估图结构是否合理、实体抽取是否准确、关系路径是否有效、社区摘要是否保真、子图选择是否覆盖问题所需证据。

目前，GraphRAG 仍缺乏统一评估框架。不同系统使用不同图构建方式、检索策略和生成流程，导致实验结果难以直接比较。未来可以从五个层面建立评估体系：一是图谱质量，包括实体准确率、关系准确率和冗余程度；二是检索质量，包括相关子图召回率和路径合理性；三是生成质量，包括答案完整性、可读性和事实一致性；四是解释质量，包括证据链是否清楚；五是任务效果，包括用户满意度、问题解决率和领域专家评价。

### 7.5 从知识库到知识治理

知识图谱在 LLM 时代的未来方向，不应只理解为“更大的知识库”或“更复杂的检索系统”。随着生成式 AI 被用于教育、科研、企业和公共服务，系统需要回答的不只是“如何生成答案”，还包括“答案依据什么”“知识从哪里来”“是否过时”“是否存在冲突”“谁有权访问”“错误如何追责”。这些问题已经超出传统信息检索范围，进入知识治理范畴。

从这个角度看，知识图谱的角色正在变化。过去，KG 主要用于语义搜索、问答和推荐；现在，它也可以作为生成式 AI 的知识组织层、证据管理层和可信约束层。GraphRAG 正是这一变化的技术表现之一。它把外部知识、图结构和生成模型连接起来，使模型生成不再只依赖参数记忆或文本相似度，而是受到结构化知识网络的支持。

未来值得关注的方向包括动态知识图谱、神经符号推理、多模态知识图谱、自动化图谱评估、面向领域的 GraphRAG，以及 KG 与 Agent 系统的结合。尤其是在 Agent 系统中，知识图谱不仅可以提供问答知识，还可以记录任务状态、工具关系、用户偏好和长期记忆。这样，KG 可能从静态知识表示工具，进一步发展为智能系统的记忆结构和行动约束机制。

## 8 结论

大语言模型的发展改变了知识获取和知识生成方式，但并没有使知识图谱失去价值。相反，LLM 的幻觉、知识过时、不可解释和领域可靠性不足，使知识图谱获得了新的应用空间。知识图谱以显式实体关系、语义约束和可追踪路径为特点，能够补足 LLM 在事实一致性、结构化检索和可解释推理方面的不足。

本文从知识表示差异、协同机制、GraphRAG 趋势和应用挑战几个方面进行了综述。知识图谱与 LLM 的关系不是简单替代，而是互补融合。KG 为 LLM 提供结构化知识和事实约束，LLM 则为 KG 构建、更新和自然语言交互提供新的能力。二者结合，使知识系统从静态存储走向动态生成，从文本检索走向结构化关联。

GraphRAG 是这一转变的代表性方向。传统 RAG 主要依赖文本片段和向量相似度，而 GraphRAG 引入实体、关系、路径、社区和子图，使检索增强生成具有更强的结构组织能力。它的意义不只是提高问答效果，而是把生成式 AI 的外部知识基础从“相似文本集合”推进到“实体关系网络”。这为企业知识库、科研知识管理和教育知识图谱等场景提供了新的技术路径。

但 KG + LLM 的结合仍面临图谱构建成本、动态更新、幻觉污染、冲突裁决和评估标准不足等挑战。未来研究需要进一步建立可靠的图谱构建流程、透明的证据追踪机制和统一的 GraphRAG 评估框架。总体而言，知识图谱在大语言模型时代的定位正在发生变化：它不再只是知识表示工具，而是可信生成式 AI 的知识组织与治理基础设施。
