# 表达式列表

表达式列表是一个通用的结构，比如`a, b, c = 1, 2, 3`这样一个语句中`a, b, c`和`1, 2, 3`就是表达式列表。gofmt采用一个通用的算法。

表达式列表有前后两个位置，下文称锚点。这两个锚点在不同的场景下有的是合法有的是非法。

| 前后都合法位置场景                               | 前合法后非法场景                                                                | 前后都非法位置场景                                                                                                                                                                                      |
| --------------------------------------- | ----------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 索引参数<br/>.e.g `[a:b:c]`中的`a:b:c`        | 赋值语句中的右边<br/>.e.g以下语句中的`1,2`<br/>1. `a, b := 1, 2`<br/>2. `a, b = 1, 2` | 以及下语句中的`a,b`:<br/>1.函数签名 `f(a,b int)`<br/>2.定义结构体类型 `struct s{a, b int}`<br/>3. 返回值`return a,b`<br/>4.变量定义 `var a,b =1,2`<br/>5.类型参数 `type A[a, b int\|float]`<br/>6.申明变量中的值部分`var x, y = a,b` |
| 函数调用参数<br/>.e.g `f(a, b, c)`中的`a, b, c` |                                                                         |                                                                                                                                                                                                |
| 组合类型初始化元素<br/>.e.g `[]int{1,2}`中的`1,2`  |                                                                         |                                                                                                                                                                                                |
| 赋值语句中的左边<br/>.e.g`a,b=1,2`中的`a,b`       |                                                                         |                                                                                                                                                                                                |
| case语句<br/>.e.g`case 1,2:`中的`1,2`       |                                                                         |                                                                                                                                                                                                |

## 都在同一行

包括两个场景：

1. 前后两个锚点都是合法，而且在同一行；

2. 相邻的两个元素在同一行。

挨个打印表达式，然后打印逗号和空格，伪代码：

```go
// expr_list 是slice，表示表达式列表
for _, x = range expr_list {
    if i > 0 {
        print(", ")
    }
    print(x)
}
```

## 不在同一行

### 缩进

1. 表达式和上一个表达式果不在同一行就会产生一个缩进；

2. 第一个锚点是合法的，如果它的行号和第一个表达式行号也不一样也会产生一个缩进。

### 换行怎么处理

常规处理，最多两行。

#### 当换行时以下几种场景会插入`\f`：

1. 换行的时候有多个表达式；

2. 当前表达式的大小大于1e6；

3. 至少一个锚点是非法的；

4. 两个锚点都是合法的，同时当前表达式不是第一个表达式，上一个表达式也没有换行。记$size$为表达式大小，记$count$从当前表达式往前数到第一个表达式开始或者从插入`\f`的表达式为止，$size>0$的表达式个数，通过一下两个步骤判断：
   
   * 调整大小：如果当前表达式是key&value表达式（比如：map&结构体），用key表达式的大小代替当前节点的大小；
   
   * 前面一个表达式的大小和当前表达式大小至少有一个大于40，下面的不等式成立：
   
   $$
   \begin{array}{l}
radio = size\div\exp((\log\prod_{i=1}^{count} size_i)/count) \\
radio<=0.4 || radio>=2.5
\end{array}
   $$

##### 不同语句插入`\f`情况

| 语句              | 说明                          | 插入`\f`场景                              |
| --------------- | --------------------------- | ------------------------------------- |
| var spec        | `var a, b = 1, 2`           | 左边的变量列表`a, b`和右边的表达式列表`1, 2`满足场景**4** |
| type parameters | `type A[a, b int] struct{}` | 变量列表`a, b`满足场景**4**                   |
