# 用户故事 source note

沉淀后的主题页：[[wiki/topics/User Stories]]

## 原始摘录

- [[问题]]
	- 相干人系
		- 业务方/客户团队（具有产品愿景信息）
			- （测试人员、产品经理、真实用户、交互设计师）
			- 职责：从业务、组织视角审视软件，编写用户故事已经优先级
		- 技术团队（具有实现、运维软件信息）
	- 有什么由来，背景
	- 解决什么问题
	- 什么时候用，和谁有关系，怎么做，输入什么、输出什么
	- 输出的内容怎么维护
- 内容：
	- 常用格式：
		- As a (角色)
		- I like to (功能)
		- So that（价值）
			- 避免变成功能的重复
			- 主要陈述如下内容
				- 可以满足某个角色的目标
				- 可以满足整体解决方案中的一个规则或者流程
				- 可以进行用户下一步
