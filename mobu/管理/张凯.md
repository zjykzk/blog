# 张凯
- 研发管理
    - 架构
        - 召开架构review会议，与会者signoff。邀请所有内部研发，产品经理，测试经理，以及跨部门对接人。
    - 排期
        - 项目启动前，有到每个人日的排期文档（包括研发，产品，测试），以及发布时间点和版本号。
    - 执行
        - 有每日SCRUM，所有checkin, 张凯，或者山峰review。每周review进度，against 排期。及时通报延期风险。
    - 发布
        - 按时发布，基础功能完善。延期不超过一周。
- 管理
    - 团队
        - 团队每周有1：1， 15分钟。
    - 平级
        - 主动和晨光，谢培，永盛，沟通技术合作，了解对方的业务和需求。
    - 上级
        - 不要给surprise, 提前充分透明沟通，充分了解上级的意图。
